## 二、与 ad-automation、material-push-tool 的对比与学习

### 1. 从 ad-automation 学习并已应用到 2.0

| 项目 | 说明 | 2.0 处理 |
|------|------|----------|
| **优雅关闭浏览器** | 先关所有页面 → 关 context → 写 Preferences 的 `exit_type`/`exited_cleanly`，避免「意外退出」提示 | ✅ 已新增 `closeBrowserGracefully(context)`，并在结束时调用 |
| **窗口置前** | 启动后 `page.bringToFront()` 便于用户操作 | ✅ 已在 `launchBrowser` 中调用（兼容无该方法时忽略） |
| **自动处理 JS 弹窗** | `page.on('dialog', ...)` 自动 accept，避免阻塞 | ✅ 已在 `launchBrowser` 中注册 dialog 监听 |

### 2. 从 material-push-tool2.0 可借鉴（2.0 现状）

| 项目 | 说明 | 2.0 现状 |
|------|------|----------|
| **DOM 稳定性** | `waitForDOMStable`、`domStableCheck`/`domStableMaxWait` 配置 | 已有 `waitForPopoverStable` 用于复制弹窗，与推广链接页面交互方式匹配；若后续发现表格/弹窗抖动可再加 DOM 稳定等待 |
| **并行与批次** | `batchDelay`、`maxConcurrent` 配置化 | 当前为全量 `Promise.allSettled` 并行，与 promotion-link 任务量匹配；若需限流可再抽成配置 |

### 3. 未照搬的差异说明

- **ad-automation 的 Chrome 启动参数**（如 `--hide-crash-restore-bubble`、`--disable-session-crashed-bubble`）：在 2.0 中已通过「优雅关闭 + 写 Preferences」缓解退出提示，若仍遇崩溃气泡可再补参数。
- **ad-automation 的守护标签页**：ad 工具保留准备页不关以维持登录；2.0 为每个任务 `context.newPage()` 且共用同一 context，首页仅用于导航与确认，当前设计已可维持登录，无需改结构。

## 三、2.0 已做的优化汇总

1. **阶段2 修改按钮精确定位**：按链接名定位行再点「修改」，避免改到原链接、避免第一个目标账户剧名错。
2. **Excel 命名区分**：`link-tasks-v2.0.xlsx`，与 1.0 分离。
3. **优雅关闭浏览器**：`closeBrowserGracefully`，减少「意外退出」提示。
4. **窗口置前 + 自动处理弹窗**：提升可操作性与稳定性。

## 四、建议的后续自测（人工）

1. 在 2.0 目录执行 `npm start`，使用 `data/link-tasks-v2.0.xlsx` 填 1～2 个任务。
2. 确认阶段2 只修改「第一个新链接」所在行，原链接的剧未被改、第一个目标账户剧名与 Excel 一致。
3. 结束后按 Enter 关闭，确认无「意外退出」类提示。
