# 阶段2「修改」按钮精确定位：思路与伪代码

## 目标

阶段2 在搜索「第一个新链接名称」后，**只点击「目标链接所在行」的「修改」按钮**，避免点到原链接所在行。

---

## 定位思路

### 原则

1. **先定行，再定按钮**：先找到「推广名称 = firstLinkName」的那一行，再在该行内找「修改」按钮。
2. **不依赖表格顺序**：不假设「第一个可用的修改 = 第一行」，避免原链接排第一时改错。
3. **兼容 Element UI 表格**：表格一般为 `table` 或 `.el-table`，行 `tbody tr`，单元格 `td` / `.cell`，操作列有「修改」按钮。

### 步骤

1. 搜索完成后，确定表格主体选择器（如 `table tbody` 或 `.el-table__body-wrapper tbody`）。
2. 在表格内找「包含且仅匹配目标链接名称」的行：
   - 优先：某列（推广名称列）的**单元格文本等于** `firstLinkName` 的那一行。
   - 若框架难以精确定位列，可退化为：**行内任意单元格文本等于** `firstLinkName` 的那一行（需避免子串误匹配）。
3. 在该行内查找「修改」按钮（且未禁用），点击。

### 选择器约定（需按实际页面微调）

- 表格行：`tbody tr` 或 `.el-table__body-wrapper tbody tr`
- 单元格：`td` 或 `td .cell`
- 推广名称列：若已知列序（如第 2 列），用 `row.locator('td').nth(1)`；否则用「行内包含 firstLinkName 的 td」。
- 行内按钮：`row.locator('button').filter({ hasText: '修改' })`，并排除 `disabled`。

## 伪代码

```text
函数 clickModifyButtonForLink(page, firstLinkName):

  1. 等待表格稳定（可选：等待 loading 消失、等待至少一行出现）

  2. 定义表格行定位器
     rows = page.locator('tbody tr')   // 或 .el-table__body-wrapper tbody tr

  3. 找到「目标行」
     方案 A（推荐）：行内某单元格精确等于 firstLinkName
       targetRow = page.locator('tbody tr').filter({
         has: page.locator('td').getByText(firstLinkName, { exact: true })
       }).first()

     方案 B：若 A 匹配不到（例如文本被截断、前后有空格），用「包含」并取第一个
       targetRow = page.locator('tbody tr').filter({
         has: page.locator('td').filter({ hasText: firstLinkName })
       }).first()

  4. 在目标行内找「修改」按钮
     modifyBtn = targetRow.locator('button').filter({ hasText: '修改' })
     若存在多个，取第一个未禁用的：
       for i in 0..modifyBtn.count()-1:
         btn = modifyBtn.nth(i)
         if not await btn.isDisabled():
           modifyBtn = btn
           break

  5. 若 targetRow 或 modifyBtn 找不到，抛出明确错误（如「未找到链接 xxx 对应的行」或「该行无可用修改按钮」）

  6. 点击 modifyBtn
  7. 等待弹窗出现
```

## 与现有流程的衔接

- **阶段2 调用处**：由原来的 `clickModifyButton(page)` 改为 `clickModifyButtonForLink(page, firstLinkName)`，入参使用当前任务的 `newLinkNames[0]`。
- **仅阶段2 使用「按链接名定位」**：阶段1 点「复制」时仍是「当前搜索结果第一条」；阶段3 点「复制」同理。只有阶段2 必须保证改的是「第一个新链接」那一行，因此只改阶段2 的「修改」点击逻辑即可。

---

## 实现要点（Playwright）

- **精确匹配**：`getByText(firstLinkName, { exact: true })` 避免子串匹配到原链接或其它链接。
- **转义特殊字符**：若 `firstLinkName` 含正则符号，用 `firstLinkName.replace(/[.*+?^${}()|[\]\\]/g, '\\$&')` 再用于 `hasText(new RegExp(...))`；若只用 `getByText(exact: true)` 则无需转义。
- **超时**：对 `targetRow` 使用 `waitFor({ state: 'visible', timeout: 5000 })` 再取其中的按钮，避免搜索未完成就点。
- **日志**：找到目标行后打日志（如「已定位到链接 xxx 所在行，正在点击该行修改按钮」），便于排查。

按上述思路实现后，即可保证「修改」一定点在该链接所在行，从而避免改错原链接、也不会出现第一个目标账户剧名仍为模板剧名的问题。

## 具体实现示例（与现有代码对接）

### 1. 新增「按链接名定位并点击修改」函数

在 `lib/automation.js` 中新增（或替换原 `clickModifyButton` 为带 linkName 的版本）：

```javascript
/**
 * 点击指定链接所在行的「修改」按钮（避免点到原链接行）
 * @param {import('playwright').Page} page
 * @param {string} linkName - 要修改的链接名称（推广名称），用于定位行
 */
async function clickModifyButtonForLink(page, linkName) {
  log(`点击链接「${linkName}」所在行的"修改"按钮`, 'STEP');

  // 表格行：Element UI 常见为 table tbody tr 或 .el-table__body-wrapper tbody tr
  const tbody = page.locator('table tbody').or(page.locator('.el-table__body-wrapper tbody'));
  let targetRow = tbody.locator('tr').filter({
    has: page.locator('td').getByText(linkName, { exact: true })
  }).first();

  try {
    await targetRow.waitFor({ state: 'visible', timeout: config.timeout.element });
  } catch (e) {
    log(`  精确匹配未找到，尝试包含匹配`, 'INFO');
    targetRow = tbody.locator('tr').filter({
      has: page.locator('td').filter({ hasText: linkName })
    }).first();
    await targetRow.waitFor({ state: 'visible', timeout: config.timeout.element });
  }

  const modifyBtn = targetRow.locator('button').filter({ hasText: '修改' });
  const count = await modifyBtn.count();
  let btnToClick = null;
  for (let i = 0; i < count; i++) {
    const btn = modifyBtn.nth(i);
    if (!(await btn.isDisabled().catch(() => true))) {
      btnToClick = btn;
      log(`  找到该行"修改"按钮（第 ${i + 1} 个，未禁用）`, 'INFO');
      break;
    }
  }
  if (!btnToClick) throw new Error(`未找到链接「${linkName}」所在行的可用"修改"按钮`);

  await btnToClick.click();
  await page.waitForTimeout(config.delay.dialogOpen);
  log(`✅ 已点击该行"修改"按钮`, 'OK');
}
```

若保留原 `clickModifyButton(page)` 用于其它场景，可保留；阶段2 改为调用 `clickModifyButtonForLink(page, firstLinkName)`。

### 2. 阶段2 调用处修改

在 `executeTask` 中，阶段2 的 Step 5 从：

```javascript
await withRetry(
  () => clickModifyButton(page),
  '点击修改按钮'
);
```

改为：

```javascript
await withRetry(
  () => clickModifyButtonForLink(page, firstLinkName),
  '点击该链接所在行的修改按钮'
);
```

并确保 `firstLinkName` 已在前面定义（当前已有：`const firstLinkName = newLinkNames[0];`）。

### 3. 选择器微调说明

- 若实际页面表格不是 `table` 而是纯 div 表格，把 `tbody` 改为实际容器，例如：  
  `page.locator('.el-table__body-wrapper tbody tr')`。
- 若「推广名称」列有固定 class（如 `.el-table__column-name`），可进一步收窄为：  
  `targetRow.locator('td.cell').filter({ hasText: linkName })` 或按列索引 `row.locator('td').nth(名称列索引)`，再配合 `getByText(linkName, { exact: true })` 避免误匹配。
- 若一行有多个「修改」按钮（例如有批量操作），当前逻辑已取「该行内第一个未禁用的修改」，一般即操作列的那个；若结构特殊，可再按「在操作列」等条件收窄。
