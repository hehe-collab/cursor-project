# 推广链接工具 2.0 创建步骤与详细内容

**说明**：本文档仅描述如何创建 promotion-link-tool2.0，**不对原 promotion-link-tool 做任何改动**。2.0 与 1.0 并存于「管理系统批量脚本工具」文件夹中。

---

## 一、目标与原则

- **原版**：`promotion-link-tool` 保持不动，代码、配置、数据路径均不修改。
- **2.0**：新建目录 `promotion-link-tool2.0`，基于原版复制后，**仅在 2.0 内**做以下改动：
  - 阶段2「修改」按钮按**链接名精确定位行**再点击，避免改到原链接、避免第一个目标账户剧名错。
  - Excel 文件名与 1.0 区分，避免共用同一文件。
  - 2.0 的包名、启动文案、可选快捷脚本等与 1.0 区分。

---

## 二、创建 2.0 的具体步骤

### 步骤 1：复制整个项目目录

- **操作**：将 `promotion-link-tool` 整目录复制一份。
- **新目录名**：`promotion-link-tool2.0`
- **位置**：与 `promotion-link-tool` 同级，即：
  - `管理系统批量脚本工具/promotion-link-tool/`  （原版，不改）
  - `管理系统批量脚本工具/promotion-link-tool2.0/` （新建）

**建议复制时排除**（可减小体积、避免共用）：

- `node_modules/`  
  - 2.0 中在步骤 3 单独执行 `npm install`。
- `browser-data/`（可选）  
  - 不复制：2.0 使用独立浏览器配置，首次运行会生成新的 `browser-data`，需重新登录。
  - 复制：2.0 与 1.0 共用登录状态，无需再登一次。

**建议复制时保留**：

- `pw-browsers/`（若存在）  
  - 复制后可共用 Playwright 浏览器，避免 2.0 再下载；若不复制，首次运行 2.0 时 `npm start` 会按需下载。
- 除上述外的所有文件：`index.js`、`config.js`、`package.json`、`create-template.js`、`test-excel.js`、`lib/*.js`、`docs/`、`data/`（若有）、`README.md` 等。

**示例命令（在「管理系统批量脚本工具」目录下执行，仅作说明）**：

```bash
# 仅复制代码与配置，不复制 node_modules 和 browser-data
cp -R promotion-link-tool promotion-link-tool2.0
rm -rf promotion-link-tool2.0/node_modules
rm -rf promotion-link-tool2.0/browser-data
```

若需共用登录状态，则不要删除 `promotion-link-tool2.0/browser-data`。

---

### 步骤 2：在 2.0 目录内安装依赖（若步骤 1 未复制 node_modules）

- **目录**：`promotion-link-tool2.0`
- **命令**：`npm install`
- **说明**：与 1.0 依赖一致（如 `package.json` 中 `playwright`、`xlsx`），仅安装到 2.0 的 `node_modules`。

---

### 步骤 3：仅修改 2.0 内的文件（以下全部在 promotion-link-tool2.0 中操作）

以下 3.1～3.6 均为**仅在 2.0 中**的修改，原版不碰。

#### 3.1 包名与版本（package.json）

- **文件**：`promotion-link-tool2.0/package.json`
- **修改**：
  - `"name": "promotion-link-tool"` → `"name": "promotion-link-tool2.0"`
  - `"description"` 可改为带「2.0」或「精确定位修改按钮」等说明，便于区分。
- **说明**：便于在 node 生态中与 1.0 区分，不影响功能。

#### 3.2 Excel 路径与浏览器数据目录（config.js）

- **文件**：`promotion-link-tool2.0/config.js`
- **修改**：
  - `excelPath: './data/link-tasks.xlsx'` → `excelPath: './data/link-tasks-v2.0.xlsx'`
  - （可选）若希望 2.0 使用独立浏览器配置且步骤 1 未复制 browser-data，可保持 `userDataDir: './browser-data'`，2.0 会使用自己目录下的 `browser-data`，与 1.0 的 `browser-data` 互不影响（因目录已不同）。
- **说明**：任务表与 1.0 分离，避免误用同一 Excel；2.0 可单独准备 `data/link-tasks-v2.0.xlsx`。

#### 3.3 模板生成脚本（create-template.js）

- **文件**：`promotion-link-tool2.0/create-template.js`
- **修改**：
  - `const outputPath = path.join(__dirname, 'data', 'link-tasks.xlsx');` → 使用 `'link-tasks-v2.0.xlsx'`
  - 控制台输出中的 `data/link-tasks.xlsx` → `data/link-tasks-v2.0.xlsx`
- **说明**：在 2.0 中执行 `npm run template` 时生成的模板与 1.0 区分。

#### 3.4 阶段2「修改」按钮精确定位（lib/automation.js）

- **文件**：`promotion-link-tool2.0/lib/automation.js`
- **修改内容**：
  1. **新增函数** `clickModifyButtonForLink(page, linkName)`  
     - 逻辑见下文「四、2.0 核心逻辑：clickModifyButtonForLink」；或直接参考 `docs/修改按钮精确定位-思路与伪代码.md` 中的「具体实现示例」。
  2. **阶段2 调用处**：在 `executeTask` 中，将「点击修改按钮」从  
     `clickModifyButton(page)`  
     改为  
     `clickModifyButtonForLink(page, firstLinkName)`。  
     - `firstLinkName` 已存在（`const firstLinkName = newLinkNames[0];`），无需新增变量。
- **说明**：原 `clickModifyButton(page)` 可保留不删（例如其他地方未使用则可保留作备用），仅阶段2 改为调用 `clickModifyButtonForLink`。

#### 3.5 启动文案（index.js，可选）

- **文件**：`promotion-link-tool2.0/index.js`
- **修改**：将启动时的标题从「TikTok推广链接复制工具 v2.0 (并行版)」改为带 2.0 标识的文案，例如「TikTok推广链接复制工具 v2.0 (精确定位版)」或「TikTok推广链接复制工具 2.0 - 修改按钮精确定位」，便于在控制台区分当前运行的是 1.0 还是 2.0。
- **说明**：仅展示用，不影响逻辑。

#### 3.6 快捷启动脚本（可选）

- **位置**：`管理系统批量脚本工具/一键启动-推广链接2.0.sh`（与 `promotion-link-tool`、`promotion-link-tool2.0` 同级）
- **内容思路**：
  - `cd` 到 `promotion-link-tool2.0` 的绝对路径或相对脚本的路径；
  - 若存在 `node_modules` 则直接 `npm start`，否则先 `npm install` 再 `npm start`；
  - 脚本可执行：`chmod +x 一键启动-推广链接2.0.sh`
- **说明**：与素材推送 2.0 的「一键启动」用法类似，便于快速打开 2.0，不修改原 1.0 任何文件。

## 四、2.0 核心逻辑：clickModifyButtonForLink（详细内容）

阶段2 需要「先定位到链接名所在行，再点该行的修改按钮」。以下为 2.0 中建议实现的逻辑概要（与 `docs/修改按钮精确定位-思路与伪代码.md` 一致）。

- **入参**：`page`（Playwright 页面）、`linkName`（要修改的推广链接名称，即 `firstLinkName`）。
- **步骤**：
  1. 用表格 body 定位器（如 `table tbody` 或 `.el-table__body-wrapper tbody`）得到所有行。
  2. **目标行**：先找「行内某单元格文本**精确等于** linkName」的行（如 `td.getByText(linkName, { exact: true })`）；若超时或找不到，再退化为「行内某单元格**包含** linkName」的行。
  3. 在该行内查找「修改」按钮（`button` 且 `hasText: '修改'`），取第一个**未禁用**的并点击。
  4. 等待弹窗出现（沿用现有 `config.delay.dialogOpen` 等）。
- **错误处理**：若找不到目标行或该行无可用「修改」按钮，抛出明确错误信息（如「未找到链接 xxx 对应的行」），便于排查。
- **日志**：建议打日志如「已定位到链接 xxx 所在行，正在点击该行修改按钮」「已点击该行修改按钮」。

具体代码可照抄或微调 `docs/修改按钮精确定位-思路与伪代码.md` 中的「具体实现示例」；若实际页面表格结构不同（如非 `table` 而为 div 表格），只需在 2.0 的 `lib/automation.js` 中微调选择器，无需改 1.0。

## 五、验收与使用

- **原版**：继续使用 `promotion-link-tool`，`npm start`、`data/link-tasks.xlsx` 不变。
- **2.0**：
  - 在 `promotion-link-tool2.0` 下执行 `npm install`（若未复制 node_modules）。
  - 执行 `npm run template` 生成 `data/link-tasks-v2.0.xlsx`（若尚未有该文件）。
  - 将任务填入 `data/link-tasks-v2.0.xlsx` 后执行 `npm start`（或通过「一键启动-推广链接2.0.sh」）。
- **验证**：用同一批任务在 2.0 跑一遍，确认：① 原链接的剧未被改；② 第一个目标账户（第一个新链接）的剧名与 Excel 中「剧名」一致，不再出现误显示为模板剧名（如「情断的人刚进-IDN」）的情况。
