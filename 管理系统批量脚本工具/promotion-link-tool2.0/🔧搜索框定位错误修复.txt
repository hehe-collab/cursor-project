❌ 问题现象
═══════════════════════════════════════════════════════════

**用户反馈**：
搜索时把模板推广链接名称输入到"推广ID"框中了，应该输入到"推广名称"框中。

**页面布局**（参考截图）：
```
推广ID: [请输入推广ID]  推广名称: [请输入推广名称]  ...  [搜索]
```

**错误行为**：
- 定位器：`input[placeholder*="推广"]`
- 使用：`.first()`
- 结果：选中了第1个输入框（推广ID）❌

**正确行为**：
- 应该选中第2个输入框（推广名称）✅

═══════════════════════════════════════════════════════════
🔍 原因分析
═══════════════════════════════════════════════════════════

### 原始代码

```javascript
// ❌ 太模糊，会匹配到"推广ID"和"推广名称"
const nameInput = page.locator('input[placeholder*="推广"]').first();
```

**问题**：
1. `placeholder*="推广"` 会匹配所有包含"推广"的输入框
   - "请输入推广ID" ✅ 匹配
   - "请输入推广名称" ✅ 匹配
   
2. `.first()` 选择第一个匹配的元素
   - 第1个：推广ID ← **被选中**
   - 第2个：推广名称

3. 结果：输入到了错误的框（推广ID）

═══════════════════════════════════════════════════════════
✅ 修复方案
═══════════════════════════════════════════════════════════

### 策略1：通过 placeholder 包含"名称"定位

```javascript
// 尝试找 placeholder 包含"名称"的输入框
nameInput = page.locator('input[placeholder*="名称"]').first();
```

**优点**：更精确
**可能的 placeholder**：
- "请输入推广名称"
- "推广名称"
- 等包含"名称"的变体

---

### 策略2：使用第2个输入框

```javascript
// 推广名称通常在推广ID后面（第2个）
const inputs = page.locator('input[placeholder*="推广"]');
nameInput = inputs.nth(1);  // 索引1 = 第2个
```

**优点**：基于页面布局逻辑
**假设**：推广名称是第2个包含"推广"的输入框

---

### 最终方案：组合策略（更稳健）

```javascript
let nameInput;

// 策略1: 尝试通过 placeholder 包含"名称"定位
try {
  nameInput = page.locator('input[placeholder*="名称"]').first();
  await nameInput.waitFor({ state: 'visible', timeout: 3000 });
  log(`  找到推广名称输入框（策略1）`);
} catch (e) {
  // 策略2: 如果策略1失败，使用第2个输入框
  const inputs = page.locator('input[placeholder*="推广"]');
  const count = await inputs.count();
  
  if (count >= 2) {
    nameInput = inputs.nth(1);  // 第2个
    log(`  使用第2个输入框（推广名称）`);
  } else {
    nameInput = inputs.first();  // 兜底
  }
}
```

**优点**：
- ✅ 优先使用更精确的定位（包含"名称"）
- ✅ 失败后回退到位置定位（第2个）
- ✅ 包含兜底逻辑
- ✅ 有详细日志便于调试

═══════════════════════════════════════════════════════════
📊 定位器对比
═══════════════════════════════════════════════════════════

| 定位方式 | 表达式 | 优点 | 缺点 |
|---------|--------|------|------|
| **原始（错误）** | `input[placeholder*="推广"].first()` | 简单 | ❌ 太模糊，选错框 |
| **策略1** | `input[placeholder*="名称"].first()` | ✅ 精确 | 如果placeholder不含"名称"会失败 |
| **策略2** | `input[placeholder*="推广"].nth(1)` | ✅ 逻辑清晰 | 依赖页面布局顺序 |
| **组合策略** | 先策略1，失败后策略2 | ✅ 稳健 | 稍复杂 |

🎓 经验教训
═══════════════════════════════════════════════════════════

### 1. 定位器要精确

**不够精确**：
```javascript
input[placeholder*="推广"]  // 太模糊，匹配多个
```

**更精确**：
```javascript
input[placeholder*="名称"]  // 更具体
input[placeholder*="推广名称"]  // 最精确
```

---

### 2. 使用 .first() 要小心

`.first()` 会选择第一个匹配的元素，如果有多个匹配：
- 第1个不一定是你要的
- 需要确保定位器足够精确

**替代方案**：
- `.nth(0)` - 第1个
- `.nth(1)` - 第2个
- `.last()` - 最后一个
- `.filter({ ... })` - 进一步过滤

---

### 3. 实际页面可能与预期不同

在开发工具时，要考虑：
- 页面布局可能变化
- Placeholder文字可能不同
- 输入框顺序可能调整

**最佳实践**：
- ✅ 使用多种策略
- ✅ 添加详细日志
- ✅ 实际测试验证

---
