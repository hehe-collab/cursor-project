❌ 问题根源
═══════════════════════════════════════════════════════════

**错误日志**：
```
browserType.launchPersistentContext: Executable doesn't exist at 
/Users/liujian/Library/Caches/ms-playwright/chromium-1208/...
```

**问题分析**：
1. ✅ 浏览器已复制到 `pw-browsers/` 目录（1.1G）
2. ✅ 浏览器文件存在且完整
3. ❌ 但Playwright还是去系统默认路径找浏览器

**根本原因**：
环境变量 `PLAYWRIGHT_BROWSERS_PATH` 的设置时机不对！

═══════════════════════════════════════════════════════════
🔍 代码对比
═══════════════════════════════════════════════════════════

### ❌ 错误的代码（原始版本）

```javascript
// lib/automation.js
const { chromium } = require('playwright');  // ← Playwright已加载
const config = require('../config');
const { log, withRetry } = require('./utils');

// ===== 配置Playwright浏览器路径 =====
process.env.PLAYWRIGHT_BROWSERS_PATH = require('path').join(__dirname, '../pw-browsers');
// ↑ 太晚了！Playwright已经读取了默认路径
```

**问题**：
- Playwright模块在 `require('playwright')` 时就初始化
- 此时会读取 `PLAYWRIGHT_BROWSERS_PATH` 环境变量
- 如果环境变量未设置，使用系统默认路径
- **后续再设置环境变量已经无效！**

---

### ✅ 正确的代码（修复后）

```javascript
// lib/automation.js
// ===== 配置Playwright浏览器路径（必须在require playwright之前）=====
const path = require('path');
process.env.PLAYWRIGHT_BROWSERS_PATH = path.resolve(__dirname, '../pw-browsers');
// ↑ 先设置环境变量

const { chromium } = require('playwright');  // ← 然后加载Playwright
const config = require('../config');
const { log, withRetry } = require('./utils');
```

**关键**：
- **必须在 `require('playwright')` 之前设置环境变量**
- 这样Playwright初始化时才能读取到正确的路径

### 关键规则

**环境变量必须在模块加载之前设置**

```javascript
// ✅ 正确的顺序
process.env.SOME_ENV = 'value';
const module = require('module-that-reads-env');

// ❌ 错误的顺序
const module = require('module-that-reads-env');
process.env.SOME_ENV = 'value';  // 太晚了！
```

### Playwright特定规则

**`PLAYWRIGHT_BROWSERS_PATH` 必须在 require playwright 之前设置**

```javascript
// ✅ 正确
process.env.PLAYWRIGHT_BROWSERS_PATH = '/path/to/browsers';
const { chromium } = require('playwright');

// ❌ 错误
const { chromium } = require('playwright');
process.env.PLAYWRIGHT_BROWSERS_PATH = '/path/to/browsers';
```

### 为什么会犯这个错误？

在创建推广链接工具时：
1. 参考了其他工具的代码
2. 但可能复制粘贴时顺序弄错了
3. 或者写代码时先写了 require 语句，后来才添加环境变量设置
4. **没有意识到顺序的重要性**
