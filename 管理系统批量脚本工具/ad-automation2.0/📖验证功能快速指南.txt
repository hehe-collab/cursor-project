📖 验证功能快速指南
====================================================================

一、功能概览
--------------------------------------------------------------------

🎯 目标：确保脚本执行内容与Excel配置完全一致

✅ 自动截图：5张关键截图/任务
✅ 增强日志：实时显示Excel vs 实际值对比
✅ 零配置：默认开启，无需额外操作


二、快速使用
--------------------------------------------------------------------

### 步骤1：正常执行脚本

```bash
npm start
```

脚本会自动：
- 📸 在关键步骤自动截图
- 🔍 在关键操作后验证数据
- ⚠️  发现不一致时立即警告

---

### 步骤2：观察终端日志

**正常情况（数据一致）**：
```
  📋 Excel配置: 账户1 = 123456789
  ✓ 实际执行: 123456789
  🔍 是否一致: ✅
```

**异常情况（数据不一致）**：
```
  📋 Excel配置: 出价 = 1.5
  ✓ 实际执行: 1.2
  🔍 是否一致: ❌
  ⚠️  数据不一致！字段"出价"，Excel为"1.5"，实际为"1.2"
```

→ 看到 ❌ 立即检查！

---

### 步骤3：执行后查看截图

```bash
# macOS
open ./screenshots/

# Windows
explorer screenshots
```

**截图说明**：
- 每个任务ID有独立文件夹
- 5张截图按时间顺序排列
- 对比截图与Excel配置是否一致

五、配置开关
--------------------------------------------------------------------

### 关闭截图（节省空间）

编辑 `config.js`：

```javascript
validation: {
  enableScreenshot: false,  // 关闭截图
  // ...
}
```

---

### 关闭验证日志（简化输出）

编辑 `config.js`：

```javascript
validation: {
  enableValidationLog: false,  // 关闭验证日志
  // ...
}
```

---

### 调整截图质量

编辑 `config.js`：

```javascript
validation: {
  screenshotQuality: 60,  // 降低到60%（更小文件）
  // 或
  screenshotQuality: 100, // 提高到100%（更高清晰度）
}
```


六、常见场景
--------------------------------------------------------------------

### 场景1：测试阶段

**建议配置**：
```javascript
validation: {
  enableScreenshot: true,     // ✅ 开启截图
  enableValidationLog: true,  // ✅ 开启验证日志
  screenshotQuality: 80,
}
```

**原因**：
- 需要详细验证脚本是否正确执行
- 出错时有完整截图证据

---

### 场景2：稳定运行阶段

**建议配置**：
```javascript
validation: {
  enableScreenshot: true,     // ✅ 开启截图
  enableValidationLog: false, // ❌ 关闭验证日志（减少输出）
  screenshotQuality: 80,
}
```

**原因**：
- 脚本已稳定，不需要实时验证日志
- 保留截图作为证据和备份

---

### 场景3：大批量任务（100+）

**建议配置**：
```javascript
validation: {
  enableScreenshot: false,    // ❌ 关闭截图（节省空间）
  enableValidationLog: false, // ❌ 关闭验证日志
  screenshotQuality: 80,
}
```

**原因**：
- 100个任务会产生500张截图（100-250MB）
- 脚本已充分测试，可以关闭验证功能

**替代方案**：
- 只为前5-10个任务开启截图
- 执行后在后台抽查实际数据

---

### 场景4：高风险任务

**建议配置**：
```javascript
validation: {
  enableScreenshot: true,     // ✅ 开启截图
  enableValidationLog: true,  // ✅ 开启验证日志
  screenshotQuality: 100,     // 🔝 最高质量
}
```

**原因**：
- 预算大、账户重要
- 需要最详细的证据和验证


七、疑难解答
--------------------------------------------------------------------

### Q1: 为什么没有生成截图？

**检查清单**：
1. 确认 `config.js` 中 `enableScreenshot: true`
2. 确认 `./screenshots/` 文件夹是否存在（会自动创建）
3. 查看终端日志是否有截图错误提示

---

### Q2: 截图文件太大，如何优化？

**方案1：降低质量**
```javascript
screenshotQuality: 60,  // 从80降到60
```

**方案2：定期清理**
```bash
# 删除7天前的截图
find ./screenshots -type f -mtime +7 -delete
```

**方案3：只保留重要任务截图**
- 执行完成后，手动删除不重要任务的文件夹

---

### Q3: 验证日志显示不一致，但实际是正确的？

**可能原因**：
1. Excel单元格格式问题（如数字被存为文本）
2. 小数点位数差异（如1.5 vs 1.50）
3. 浮点数精度问题（如1.1 vs 1.10000001）

**解决方法**：
- 检查Excel单元格格式
- 查看截图确认实际值
- 如确认正确，可忽略该警告

---

### Q4: 如何批量查看截图？

**macOS**：
```bash
# 幻灯片模式查看某个任务的所有截图
qlmanage -p ./screenshots/001/*.png
```

**Windows**：
- 打开文件夹，选中所有截图
- 双击第一张，使用箭头键翻页
