三、观察重点
--------------------------------------------------------------------

### ✅ 正常运行的标志

1. **日志显示"并行模式已启用"**
   - 如果显示"串行模式"，说明config.js没改对

2. **同时打开多个标签页**
   - 看到5个Chrome标签页同时打开
   - 每个标签页执行不同任务

3. **任务同时执行**
   - 日志显示多个"开始"消息几乎同时出现
   - 不是一个接一个

4. **批次统计正确**
   - 每批结束显示成功数量
   - 批次耗时显示

5. **总耗时大幅缩短**
   - 10个任务：约5-6分钟（vs 串行25分钟）
   - 5个任务：约2.5-3分钟（vs 串行12-15分钟）

---

### ⚠️ 需要注意的情况

1. **内存占用增加**
   - 正常（5个标签页同时运行）
   - Mac Mini M4 Pro完全支持

2. **浏览器可能卡顿片刻**
   - 5个标签页同时操作时正常
   - 不影响任务执行

3. **批次间有2秒延迟**
   - 这是正常的（避免服务器压力）
   - 不是bug

---

### 性能验证方法

**测试对比**（可选）：

1. **串行模式测试**（基准）
   ```javascript
   // config.js
   parallel: { enabled: false }
   ```
   运行5个任务，记录耗时：约12-15分钟

2. **并行模式测试**（对比）
   ```javascript
   // config.js
   parallel: { enabled: true }
   ```
   运行相同5个任务，记录耗时：约2.5-3分钟

3. **计算提升**
   ```
   提升 = (串行耗时 - 并行耗时) / 串行耗时 × 100%
        = (12.5 - 2.5) / 12.5 × 100%
        = 80%
   ```


五、故障排查速查表
--------------------------------------------------------------------

### 问题：日志显示"串行模式"

**原因**：config.js 的 parallel.enabled 没有改为 true

**解决**：
```javascript
// config.js 第40行
enabled: true,  // 确认是true，不是false
```

---

### 问题：只打开1个标签页

**原因**：并行模式没有生效

**检查**：
1. config.js 是否已保存
2. 是否重启了脚本（Ctrl+C 退出，重新 npm start）

---

### 问题：某个任务卡住很久

**原因**：单个任务遇到问题

**处理**：
1. 等待（有10分钟超时保护）
2. 超时后会自动失败，不影响其他任务
3. 查看终端日志了解失败原因

---

### 问题：浏览器很卡

**原因**：并发数可能过高

**解决**：
```javascript
// config.js
parallel: {
  enabled: true,
  maxConcurrent: 3,  // 降低到3
  batchDelay: 3000,  // 增加延迟
}
```

---

### 问题：多个任务同时失败

**原因**：服务器限流

**解决**：
```javascript
// config.js
parallel: {
  enabled: true,
  maxConcurrent: 3,     // 降低并发
  batchDelay: 5000,     // 增加延迟到5秒
}
```
