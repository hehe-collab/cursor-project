# TikTok 广告批量建设自动化脚本

> 基于 Playwright 的浏览器自动化脚本，用于批量创建 TikTok 广告任务

## 📋 功能特点

- ✅ 批量选择主体和账户
- ✅ 自动设置推广项目（Pixel、项目名称、Smart+、启用开关）
- ✅ 自动设置广告组（推广链接、素材、标题、优化目标、出价、预算、开始时间等）
- ✅ 支持重复提交（同一内容提交多次，间隔2秒）
- ✅ Excel 数据驱动，支持任务分组和配置继承
- ✅ 智能错误处理，失败时暂停手动干预
- ✅ 跨平台兼容（macOS / Windows / Linux）

## 🚀 快速开始

### 1. 环境要求

- **Node.js**: v16 或更高版本
- **操作系统**: macOS / Windows / Linux
- **浏览器**: 自动安装 Chromium（无需手动安装）

### 2. 安装依赖

```bash
# 进入项目目录
cd ad-automation

# 安装依赖（首次使用）
npm install
```

### 3. 生成 Excel 模板

```bash
# 生成 data/tasks-v2.0.xlsx 模板文件
npm run template
```

### 4. 填写任务数据

打开 `data/tasks-v2.0.xlsx`，填写任务信息：

| 任务 | 主体 | 账户ID | Pixel | 项目名称 | 推广链接关键词 | 素材关键词 | 标题 | 优化目标 | 出价 | 预算 | 开始日期 | 开始时间 | 提交次数 |
|------|------|--------|-------|----------|---------------|-----------|------|---------|------|------|---------|---------|---------|

| 1 |  | 7595500599618469889 |  |  | C-xd-IDN-父女情深-3.0V |  |  |  |  |  |  |  |  |
| 1 |  | 7595500590164246544 |  |  | C-xd-IDN-父女情深-5.0V |  |  |  |  |  |  |  |  |

- 相同**任务编号**的行会合并为一次批量提交

### 5. 运行脚本

```bash
npm start
```

**执行流程**：
1. 脚本启动后，浏览器会自动打开并导航到批量工具页面
2. **如果需要登录**，请在浏览器中手动登录
3. **关闭所有弹窗**（如"更改密码"提示）
4. 确认在批量工具页面后，**按 Enter 键**开始执行
5. 脚本会自动完成所有步骤
6. 如果遇到错误，会暂停等待手动处理，处理完后按 Enter 继续

### 素材检索说明 🎨

**素材关键词**和**素材ID**二选一，素材关键词优先级更高：

| 填写情况 | 实际使用 | 说明 |
|---------|---------|------|
| ✅ 素材关键词有值 | 使用关键词检索 | 推荐方式，简单快捷 |
| ✅ 素材ID有值 | 使用ID检索 | 精确选择特定素材 |
| ✅ 两者都有值 | 使用关键词检索 | 素材ID被忽略 |
| ❌ 两者都没值 | 错误 | 该账户会被跳过 |

**素材ID填写方法**：
1. 从素材选择界面复制素材ID
2. 粘贴到Excel的"素材ID"单元格
3. 多个ID用换行分隔（在单元格内按 `Alt+Enter` 或 `Option+Enter`）
4. 脚本会自动处理分号、换行符等特殊字符

**示例**（在Excel单元格中）：
```
7601467303255392264
7601467161916375047
7601468503409344520
```

详细说明请参考：`素材ID检索使用指南.txt`

### 时间格式说明 ⏰

**开始时间**支持多种格式，脚本会自动识别：

| 输入格式 | 说明 | 示例 | 自动转换为 |
|---------|------|------|-----------|
| **HH:MM:SS** | 完整时间格式（推荐） | 02:00:00 | 02:00:00 |
| **HH:MM** | 简化格式 | 02:00 | 02:00:00 |
| **小数** | Excel 时间序列号 | 0.041667 | 01:00:00 |

**注意事项**：
1. ✅ **推荐格式**：使用 `HH:MM:SS` 或 `HH:MM`（如 `02:00:00` 或 `02:00`）
2. ⚠️ **Excel 自动格式化**：如果你在 Excel 中输入时间，Excel 可能将其存储为小数，脚本会自动识别并转换
3. ✅ **设置 Excel 单元格格式**：
   - 选中"开始时间"列
   - 右键 → 设置单元格格式
   - 选择"时间"类型
   - 选择格式：`13:30:55` (HH:MM:SS)
4. ⚠️ **避免日期时间混合**：确保"开始时间"列只包含时间，不要包含日期部分

**常见错误示例**：
- ❌ `2026-02-10 02:00:00` （包含日期，应分开填写）
- ❌ `2:00` （应补零为 `02:00`）
- ✅ `02:00:00` （正确）
- ✅ `02:00` （正确，自动补充为 02:00:00）

### 字段继承规则

**首行（主配置行）**：必须填写所有必填字段
**后续行**：
- **必须填写**：`账户ID`、`推广链接关键词`、`标题`（每个账户独立）
- **可以留空**：其他字段自动继承首行配置
- **可以覆盖**：`Pixel`、`标题`、`启用` 等字段可以逐行自定义

**示例**：
```
任务 | 主体 | 账户ID         | Pixel    | 项目名称      | 链接关键词      | 标题         | 启用
-----|------|---------------|----------|--------------|----------------|-------------|------
  1  | MAD  | 7561696938... | IDNLow-X | 父女情深-IDN | ...2.5V        | hsx-7,hsx-8 | true
  1  |      | 7595500599... |          |              | ...3.0V        | hmj-1,hmj-2 |      (继承true)
  1  |      | 7595500590... | THLow-X  |              | ...5.0V        | ph-1,ph-2   | false (覆盖)
```

**说明**：
- 第1个账户使用标题 `hsx-7, hsx-8`
- 第2个账户使用标题 `hmj-1, hmj-2`（不同）
- 第3个账户使用标题 `ph-1, ph-2`（不同）并覆盖 Pixel 和启用状态

- 上传到 OneDrive / 百度网盘 / iCloud

**方法 C: Git**

```bash
git init

git commit -m "initial commit"

git clone <仓库地址>
```

```powershell
# 打开 PowerShell 或 CMD
cd D:\projects\ad-automation
```

## 🐛 常见问题解决

### 问题1: npm install 失败

**Windows 权限错误**：
```powershell
# 以管理员身份运行 PowerShell
# 右键 PowerShell → "以管理员身份运行"
```

**PowerShell 执行策略错误**：
```powershell
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
```

**网络问题（下载慢）**：
```bash
# 使用国内镜像
npm config set registry https://registry.npmmirror.com
npm install
```

### 问题2: Playwright 浏览器下载失败

```bash
# 方法1: 手动安装浏览器
npx playwright install chromium

# 方法2: 使用国内镜像
# macOS/Linux
export PLAYWRIGHT_DOWNLOAD_HOST=https://npmmirror.com/mirrors/playwright
npm install

# Windows PowerShell
$env:PLAYWRIGHT_DOWNLOAD_HOST="https://npmmirror.com/mirrors/playwright"
npm install
```

### 问题3: 找不到"设置"按钮

**原因**：页面加载未完成或弹窗遮挡

**解决**：
1. 确认已关闭所有弹窗（密码修改提示、通知等）
2. 确认在批量工具页面
3. 手动点击"设置"按钮后，按 Enter 继续

### 问题4: 素材选择不正确

**原因**：搜索结果不匹配关键词

**解决**：
1. 脚本会在选择前验证搜索结果
2. 如果结果不匹配，会暂停提示确认
3. 检查 Excel 中的"素材关键词"是否正确
4. 手动选择正确的素材后，按 Enter 继续

### 问题5: 标题找不到或滚动失败

**原因**：标题列表很长，需要滚动查找

**解决**：
1. 脚本会自动滚动查找（最多30次）
2. 如果找不到，会暂停提示
3. 手动在下拉列表中滚动并选择标题
4. 按 Enter 继续

### 问题6: 时间设置卡住

**原因**：日期时间选择器有两个确定按钮

**解决**：
- 脚本已自动处理，会依次点击时间层和日期层的确定按钮
- 如果仍然失败，手动点击确定按钮后按 Enter

### 问题7: 提交按钮找不到

**原因**：提交按钮可能被遮挡或位置不在视口

**解决**：
1. 脚本会自动滚动到提交按钮
2. 如果失败，会保存 `debug-submit.png` 截图
3. 手动点击提交按钮后，按 Enter 继续

### 问题8: 登录状态丢失

**原因**：`browser-data` 文件夹损坏或被删除

**解决**：
```bash
# 删除旧的浏览器数据
rm -rf browser-data   # macOS/Linux
rmdir /s browser-data # Windows

# 重新运行，会自动创建新的浏览器数据
npm start
# 手动登录一次，保存登录状态
```

## 🎯 执行流程详解

### 单个任务的执行步骤

1. **选择主体** → 在主体下拉框中选择（如 MAD）
2. **选择账户** → 搜索并选择多个账户ID
3. **点击项目"设置"** → 打开项目设置弹窗
4. **设置推广项目**：
   - 选择 Pixel（每个账户独立设置）
   - 填写项目名称（每个账户独立设置）
   - 开启 Smart+（每个账户独立设置）
   - 设置启用开关（每个账户独立设置）
   - 点击确定关闭弹窗
5. **点击广告组"设置"** → 打开广告组设置弹窗
6. **设置广告组**（每个账户独立设置）：
   - 选择推广链接（搜索关键词）
   - 选择素材（搜索关键词，全选30条）
   - 选择标题（多选，支持滚动查找）
   - 选择优化目标（价值/下单）
   - 输入出价和预算
   - 选择开始日期和时间
   - 可选：选择年龄、认证身份、商品库、商品
   - 点击确定关闭弹窗
7. **提交任务** → 点击提交按钮
8. **重复提交**（如果设置了提交次数>1）：
   - 等待2秒
   - 再次点击提交按钮
   - 重复直到达到指定次数

### 多任务批处理

```
任务1：
  - 设置内容
  - 提交 N1 次（间隔2秒）
  
→ 刷新页面 →

任务2：
  - 设置内容
  - 提交 N2 次（间隔2秒）
  
→ 刷新页面 →

...
```
