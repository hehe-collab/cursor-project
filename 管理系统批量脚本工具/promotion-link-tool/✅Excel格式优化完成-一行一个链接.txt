## 📊 格式变更对比

### 原格式（一个单元格内多行）

| 任务ID | 模板链接名称 | 剧名 | 集数 | 新链接名称列表 | 备注 |
|--------|-------------|------|------|----------------|------|
| 1 | 模板1 | 剧名1 | 9 | 链接1\n链接2\n链接3\n链接4 | |

**特点**：
- 一个任务占1行
- 所有新链接挤在一个单元格，用\n分隔

---

### 新格式（一行一个链接）⭐

| 任务ID | 模板链接名称 | 剧名 | 集数 | 新链接名称 | 备注 |
|--------|-------------|------|------|-----------|------|
| 1 | 模板1 | 剧名1 | 9 | 链接1 | 备注1 |
| 1 | | | | 链接2 | |
| 1 | | | | 链接3 | |
| 1 | | | | 链接4 | |

**特点**：
- ✅ 每个新链接占一行
- ✅ 第一行填完整信息
- ✅ 后续行只填任务ID和新链接
- ✅ 更清晰，符合习惯

## ✅ 修改的文件

### 1. `lib/excel.js` - 数据读取逻辑（重要）

**核心改动**：
- ✅ 字段名：`新链接名称列表` → `新链接名称`
- ✅ 读取逻辑：按任务ID分组
- ✅ 每行收集一个新链接名称
- ✅ 同一任务的模板/剧名/集数取第一个非空值
- ✅ 移除 `cleanLinkNames` 导入（不再需要）

**关键代码**：
```javascript
// 按任务ID分组
const groupedByTaskId = {};

data.forEach(row => {
  const taskId = String(row['任务ID']);
  
  // 第一次出现，创建任务对象
  if (!groupedByTaskId[taskId]) {
    groupedByTaskId[taskId] = {
      taskId: taskId,
      templateLinkName: null,
      dramaName: null,
      freeEpisodes: null,
      newLinkNames: [],
      remark: ''
    };
  }
  
  const task = groupedByTaskId[taskId];
  
  // 收集非空字段（取第一个非空值）
  if (row['模板链接名称'] && !task.templateLinkName) {
    task.templateLinkName = row['模板链接名称'];
  }
  
  // 收集新链接名称（每行都收集）
  task.newLinkNames.push(row['新链接名称']);
});

// 转换为数组
const tasks = Object.values(groupedByTaskId);
```
