❌ 问题现象
═══════════════════════════════════════════════════════════

**错误日志**：
```
[STEP] 输入新链接名称: C-xd-IDN-落叶的告别-印尼-2.5W-S+xd-1-S+tr
[ERROR] ❌ 输入第一个新链接名称 失败（已重试3次）: 
locator.waitFor: Timeout 10000ms exceeded.
  - waiting for locator('.el-dialog').filter({ hasText: /复制|广告/ })
```

**实际情况**：
- ✅ "复制"按钮已成功点击
- ✅ 弹窗已经打开（截图可见）
- ❌ 但代码无法定位到弹窗
- 原因：弹窗标题不包含"复制"或"广告"文字

═══════════════════════════════════════════════════════════
🔍 原因分析
═══════════════════════════════════════════════════════════

### 原始代码（错误）

```javascript
// ❌ 依赖弹窗标题包含特定文字
const dialog = page.locator('.el-dialog')
  .filter({ hasText: /复制|广告/ })
  .first();
```

**问题**：
1. 假设弹窗标题包含"复制"或"广告"
2. 实际弹窗标题可能是：
   - "请输入一键按2折，一按行拼图"
   - 或其他文字
3. `.filter({ hasText: ... })` 找不到 → 超时

**教训**：
- ❌ 不要假设弹窗内容
- ❌ 不要依赖可变的文字内容
- ✅ 使用更通用的定位方式

═══════════════════════════════════════════════════════════
✅ 修复方案
═══════════════════════════════════════════════════════════

### 方案1：使用 `:visible` 伪类（推荐）

```javascript
// ✅ 找到当前可见的弹窗，不依赖文字内容
const dialog = page.locator('.el-dialog:visible').first();
```

**优点**：
- ✅ 不依赖弹窗标题或内容
- ✅ 只要弹窗可见就能找到
- ✅ 简单可靠

---

### 方案2：等待任意弹窗出现

```javascript
// ✅ 等待页面上出现任意 .el-dialog
await page.waitForSelector('.el-dialog', { state: 'visible' });
const dialog = page.locator('.el-dialog').first();
```

**优点**：
- ✅ 明确等待弹窗出现
- ✅ 不依赖内容

---

### 输入框定位优化

```javascript
// 策略1: 尝试通过 placeholder 定位
nameInput = dialog.locator('input[placeholder*="推广"]')
  .or(dialog.locator('input[placeholder*="名称"]'))
  .or(dialog.locator('input[placeholder*="链接"]'))
  .first();

// 策略2: 兜底使用弹窗内第一个 input
if (策略1失败) {
  nameInput = dialog.locator('input').first();
}
```

**优点**：
- ✅ 多种 placeholder 匹配
- ✅ 有兜底方案

💡 Element UI 弹窗最佳实践
═══════════════════════════════════════════════════════════

### Element UI 弹窗结构

```html
<div class="el-dialog__wrapper" style="display: block;">
  <div class="el-dialog" style="...">
    <div class="el-dialog__header">
      <span class="el-dialog__title">标题</span>
    </div>
    <div class="el-dialog__body">
      <!-- 内容，包含输入框 -->
      <input ...>
    </div>
    <div class="el-dialog__footer">
      <button>取消</button>
      <button>确定</button>
    </div>
  </div>
</div>
```

### 推荐的定位方式

**1. 定位弹窗本身**
```javascript
// ✅ 推荐：使用 :visible
.el-dialog:visible

// ⚠️ 备用：使用 wrapper
.el-dialog__wrapper[style*="display: block"] .el-dialog

// ❌ 不推荐：依赖标题文字
.el-dialog.filter({ hasText: '特定文字' })
```

**2. 定位弹窗内的元素**
```javascript
// ✅ 先找到弹窗，再找内部元素
const dialog = page.locator('.el-dialog:visible');
const input = dialog.locator('input');
const confirmBtn = dialog.locator('button').filter({ hasText: '确定' });
```

**3. 等待弹窗出现**
```javascript
// ✅ 明确等待
await page.waitForSelector('.el-dialog', { state: 'visible' });
```

🎓 经验总结
═══════════════════════════════════════════════════════════

### 1. 不要假设UI文字内容

**反例**：
```javascript
// ❌ 假设标题是"复制"
.filter({ hasText: '复制' })
```

**问题**：
- UI文字可能改变
- 不同页面可能不同
- 多语言环境会失败

**最佳实践**：
- ✅ 使用结构性定位（类名、可见性）
- ✅ 文字只用于最终确认（如按钮）

---

### 2. 使用 :visible 伪类很有用

Playwright支持CSS伪类：
```javascript
:visible   // 可见的元素
:hidden    // 隐藏的元素
:enabled   // 启用的元素
:disabled  // 禁用的元素
```

**弹窗场景特别适合**：
```javascript
// ✅ 找到当前打开的弹窗
.el-dialog:visible

// ✅ 排除隐藏的弹窗
```

---

### 3. 多策略 + 兜底

```javascript
try {
  // 策略1: 最精确的
} catch {
  try {
    // 策略2: 稍宽松的
  } catch {
    // 策略3: 兜底的
  }
}
```

**优点**：
- ✅ 适应不同页面变化
- ✅ 提高成功率
- ✅ 有详细日志便于调试
