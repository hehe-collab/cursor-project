## 🚀 使用方法

### 1. 生成Excel模板
```bash
cd "/Volumes/存钱罐/cursor项目/promotion-link-tool"
npm run template
```

### 2. 填写任务数据
打开 `data/link-tasks.xlsx` 填写：

| 字段 | 必填 | 说明 | 示例 |
|------|-----|------|------|
| 任务ID | ✅ | 任务标识 | 1, 2, 3... |
| 模板链接名称 | ✅ | 要复制的模板 | C-xd-IDN-夜不能寐-2.5W-S+xd-18-S+tr |
| 剧名 | ✅ | 短剧名称 | 落叶的告别-印尼 |
| 免费/预览集数 | ✅ | 集数（必须>0） | 9 |
| 新链接名称列表 | ✅ | 链接名称（换行分隔） | C-xd-IDN-落叶...-1<br>C-xd-IDN-落叶...-2 |
| 备注 | ⭕ | 可选备注 | 第一批任务 |

**重要提示**：
- ⚠️ 除了"备注"，所有字段都是必填！
- ⚠️ 如果有空白字段，工具会在启动时报错并提示


### 4. 启动工具
```bash

```

1. ✅ 验证Excel数据（必填字段检查）

## 💡 核心特性

### 1. 智能工作流
```
步骤1: 搜索模板链接
       ↓
步骤2: 复制创建第1个新链接
       ↓
步骤3: 修改剧名和集数
       ↓
步骤4: 批量复制创建剩余N-1个链接（一次性输入所有名称）
```

**效率优势**：
- ✅ 只修改一次参数（不是N次）
- ✅ 批量复制（不是循环N-1次）
- ✅ 并行执行（不是串行等待）

### 2. 数据验证机制
```
读取Excel → 验证必填字段 → 验证数据格式 → 解析任务
                ↓ 发现错误
            精确定位 + 友好提示 + 立即终止
```

**验证项目**：
- ✅ 所有必填字段不能为空
- ✅ 免费/预览集数必须是正整数
- ✅ 新链接名称列表至少包含1个链接
- ✅ 精确显示错误的行号和任务ID

### 3. 并行执行架构
```
主页面（用于登录验证）
    ↓
为每个任务创建独立页面
    ↓
Promise.allSettled 并行执行
    ↓
结果汇总 + 统计
```

### 4. 步骤级重试机制
- 每个关键步骤自动重试最多3次
- 失败后暂停，等待用户干预
- 详细日志记录每次尝试

```javascript
// 必填字段列表
const requiredFields = [
  { field: '任务ID', name: '任务ID' },
  { field: '模板链接名称', name: '模板链接名称' },
  { field: '剧名', name: '剧名' },
  { field: '免费/预览集数', name: '免费/预览集数' },
  { field: '新链接名称列表', name: '新链接名称列表' },
];

// 验证每一行数据
for (let i = 0; i < data.length; i++) {
  const row = data[i];
  const rowNum = i + 2;
  const rowErrors = [];

  // 检查必填字段
  requiredFields.forEach(({ field, name }) => {
    if (value === undefined || value === null || String(value).trim() === '') {
      rowErrors.push(`"${name}" 为空`);
    }
  });

  // 验证集数必须是正整数
  if (episodes <= 0) {
    rowErrors.push(`"免费/预览集数" 必须是大于0的数字`);
  }
}

// 如果有错误，报错并终止
if (errors.length > 0) {
  throw new Error(`发现 ${errors.length} 个数据错误，请修正后重新运行`);

```

## 📋 错误处理

### 数据验证错误示例
```
❌ Excel数据验证失败！

❌ 第4行（任务ID: 3）: "免费/预览集数" 为空
❌ 第5行（任务ID: 4）: "剧名" 为空

提示：除了"备注"列，其他所有列都是必填项！
```

### 执行错误处理
- 步骤失败 → 自动重试（最多3次）
- 重试失败 → 暂停，等待用户干预
- 用户处理 → 按Enter继续下一个任务
- 最终统计 → 显示成功/失败数量
