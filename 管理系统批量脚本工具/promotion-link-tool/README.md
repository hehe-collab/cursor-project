# 🎉 TikTok推广链接复制工具 v2.0

自动化批量创建TikTok推广链接的工具。通过"复制模板 → 修改参数 → 批量复制"的方式，快速为不同短剧创建多个推广链接。

## ✨ 核心功能

- ✅ 从模板链接复制创建新链接
- ✅ 自动修改剧名、免费集数、预览集数
- ✅ 批量复制已修改好的链接（一次性创建多个）
- ✅ **🚀 并行执行多个任务（v2.0新增）**
- ✅ 步骤级自动重试机制
- ✅ 详细日志输出
- ✅ 失败暂停等待用户干预

## 🚀 v2.0 新特性

### 并行执行

- **多页面并行**：为每个任务创建独立的浏览器页面
- **效率提升**：3个任务同时执行，总耗时接近单个任务时间
- **智能调度**：使用 Promise.allSettled 确保所有任务都执行完成
- **结果汇总**：自动统计成功/失败任务数量和总耗时

**性能对比**：
```
v1.0 串行执行：任务1(9s) → 任务2(9s) → 任务3(9s) = 27秒
v2.0 并行执行：任务1、2、3 同时执行 = ~10秒
```

**提速：60-70%** ⚡️

## 🚀 快速开始

### 1. 安装依赖

```bash
cd /Volumes/存钱罐/cursor项目/promotion-link-tool
npm install
```

### 2. 生成Excel模板

```bash
npm run template
```

生成的文件：`data/link-tasks.xlsx`

### 3. 填写任务数据

打开 `data/link-tasks.xlsx`，填写：

| 字段 | 说明 | 示例 |
|------|------|------|
| 任务ID | 任务标识（可选） | 1, 2, 3... |
| 模板链接名称 | 要复制的模板链接 | 淮金之5W-S+xd-1-5+tr |
| 剧名 | 短剧名称 | 淮村的贫苦 |
| 免费/预览集数 | 免费和预览集数（相同） | 5 |
| 新链接名称列表 | 要创建的链接名称（每行一个） | C-xd-IDN-淮村的贫苦印尼-2<br>C-xd-IDN-淮村的贫苦印尼-3 |
| 备注 | 可选备注 | 第一部剧 |

**重要说明**：
- **新链接名称列表**中，第1个链接会从模板复制并修改参数
- 第2-N个链接会从第1个（已修改好）的链接复制
- 这样只需修改一次参数，大幅提升效率！

### 4. 测试数据

```bash
npm test
```

### 5. 启动工具

```bash
npm start
```

### 6. 执行流程

1. ✅ 查看任务清单
2. ✅ 浏览器自动打开推广链接页面
3. ⚠️ **首次需要手动登录**（登录状态会保存）
4. ✅ 按 **Enter** 开始执行
5. ✅ 观察自动执行过程
6. ✅ 完成后查看统计结果
7. ✅ 按 **Enter** 关闭浏览器

## 📊 执行逻辑

### 单个任务流程

假设 `新链接名称列表` 有4个链接：
```
C-xd-IDN-淮村的贫苦印尼-2
C-xd-IDN-淮村的贫苦印尼-3
C-xd-IDN-淮村的贫苦印尼-4
C-xd-IDN-淮村的贫苦印尼-5
```

**阶段1：从模板创建第一个新链接**
1. 搜索模板链接
2. 点击"复制"
3. 输入第1个链接名称
4. 确定

**阶段2：修改第一个新链接的参数**
5. 搜索第1个新链接
6. 点击"修改"
7. 修改剧名、免费集数、预览集数
8. 确定

**阶段3：批量复制第一个新链接**
9. 点击第1个链接的"复制" → 创建第2个链接
10. 点击第1个链接的"复制" → 创建第3个链接
11. 点击第1个链接的"复制" → 创建第4个链接

✅ 完成！创建了4个推广链接

## ⚙️ 配置说明

### 延迟配置（config.js）

```javascript
delay: {
  afterSearch: 1000,        // 搜索后等待
  afterCopy: 800,           // 复制后等待
  afterModify: 1000,        // 修改后等待
  dialogOpen: 500,          // 弹窗打开等待
  inputFill: 200,           // 输入框填充等待
}
```

### 重试配置

```javascript
retry: {
  enabled: true,            // 启用重试
  maxAttempts: 3,           // 最多重试3次
  retryDelay: 1000,         // 重试间隔
}
```

## 💡 使用技巧

### 技巧1：先测试再执行
```bash
npm test  # 验证Excel数据
npm start # 启动工具
```

### 技巧2：合理安排链接数量
- 建议每个任务创建10-20个链接
- 链接过多可能导致操作时间过长

### 技巧3：观察日志
- ✅ 绿色 [OK]：成功
- ⚠️ 黄色 [WARN]：警告
- ❌ 红色 [ERROR]：错误（会自动重试）

### 技巧4：处理失败
- 工具会自动重试3次
- 重试失败后会暂停，等待手动处理
- 处理完成后按 Enter 继续

## 🛠️ 故障排除

### 问题1：浏览器无法启动
**解决**：检查浏览器是否存在
```bash
ls pw-browsers/
```

### 问题2：找不到元素
**解决**：
- 检查页面是否已加载完成
- 查看终端错误日志
- 手动执行一次，观察页面元素

### 问题3：登录状态丢失
**解决**：
```bash
rm -rf browser-data/
npm start  # 重新登录
```

## ⚠️ 注意事项

1. **首次使用需要手动登录**
   - 登录状态会保存在 `browser-data/`
   - 下次启动自动登录

2. **新链接名称列表格式**
   - 每行一个链接名称
   - 第1个会被修改参数
   - 第2-N个复制第1个

3. **一个任务处理一个短剧**
   - 多个短剧创建多个任务
   - 后续可升级为并行模式

4. **浏览器共享**
   - 与广告工具、素材工具共享浏览器
   - 登录状态互通
