🐛 问题描述
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

错误信息：
  [23:28:27] ⚠️  选择素材失败: 
  错误: locator.fill: Timeout 30000ms exceeded.
  Call log:
    - waiting for locator('.el-dialog:visible')
      .filter({ hasText: '选择素材' })
      .locator('input[placeholder*="素材ID"], input[placeholder*="素材 ID"]')
      .first()

问题原因：
  1. 原有的 placeholder 选择器不够精确
  2. 素材选择弹窗中有多个输入框（素材名称、素材ID、视频ID）
  3. 使用 .first() 可能选中了错误的输入框或找不到输入框


🔧 解决方案
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

实现了三层定位策略，按优先级依次尝试：

策略 1: 表单结构定位（最可靠）
────────────────────────────────
  - 查找弹窗中所有的表单输入框（.el-form-item input）
  - 选择第二个输入框（素材名称是第一个，素材ID是第二个）
  - 优点：基于页面结构，稳定可靠
  
  代码：
    const allInputs = materialDialog.locator('.el-form-item input.el-input__inner, .el-form-item textarea');
    idInput = allInputs.nth(1);


策略 2: JavaScript 精确匹配（最智能）
────────────────────────────────────
  - 使用 evaluate 在浏览器中执行 JavaScript
  - 遍历所有输入框，检查 placeholder 或 label
  - 找到包含"素材"和"ID"的输入框
  - 给找到的输入框添加标记属性：data-material-id-input="true"
  - 然后用 Playwright 定位这个标记
  - 优点：最精确，不依赖位置
  
  代码：
    await materialDialog.evaluate(() => {
      const inputs = Array.from(document.querySelectorAll('input, textarea'));
      for (let i = 0; i < inputs.length; i++) {
        const input = inputs[i];
        const placeholder = input.getAttribute('placeholder') || '';
        const label = input.labels && input.labels[0] ? input.labels[0].textContent : '';
        
        if ((placeholder.includes('素材') && placeholder.includes('ID')) ||
            (label.includes('素材') && label.includes('ID'))) {
          input.setAttribute('data-material-id-input', 'true');
          return { found: true, placeholder, label, index: i };
        }
      }
      return { found: false };
    });
    
    idInput = materialDialog.locator('[data-material-id-input="true"]');


策略 3: 兜底方案（最简单）
────────────────────────────
  - 直接选择弹窗中的第二个 input 元素
  - 不管元素类型和位置，简单粗暴
  - 优点：总能选中一个输入框
  
  代码：
    idInput = materialDialog.locator('input').nth(1);


📊 执行逻辑
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

流程：
  1. 尝试策略1（表单结构定位）
     ├─ 成功 → 使用该输入框
     └─ 失败 → 继续策略2
  
  2. 尝试策略2（JavaScript精确匹配）
     ├─ 成功 → 使用该输入框
     └─ 失败 → 继续策略3
  
  3. 尝试策略3（兜底方案）
     ├─ 成功 → 使用该输入框
     └─ 失败 → 抛出错误，暂停脚本
