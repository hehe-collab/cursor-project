二、根本原因分析
--------------------------------------------------------------------

### 核心问题：Vue响应式数据未同步

```
┌──────────────────────────────────────────────────────────┐
│              Element UI 表单工作机制                      │
└──────────────────────────────────────────────────────────┘

1. Playwright填入：
   input.value = "7603...\n7603...\n..."   ← DOM值改变

2. 触发事件：
   input.dispatchEvent('input')
   input.dispatchEvent('change')
   input.blur()

3. Vue捕获事件：
   Element UI的事件监听器被触发

4. Vue更新响应式数据：
   ⚠️ 这一步在第1次时失败了！
   formData.materialIds = "7603...\n7603...\n..."

5. 点击搜索按钮，发送请求：
   axios.post('/api/materials/search', {
     materialIds: formData.materialIds  ← 第1次是空的！
   })

6. 后端返回：
   - 如果materialIds为空 → 返回默认30个素材
   - 如果materialIds有值 → 返回匹配的7个素材
```

---

### 为什么第1次Vue响应式数据未同步？

**原因1：Vue组件初始化未完成**（最可能 80%）
```javascript
// 第1次打开弹窗
弹窗组件mounted() {
  // 这里还在初始化响应式数据
  this.formData = {
    materialName: '',
    materialIds: ''  ← 初始化为空
  }
  
  // 绑定事件监听器
  this.$watch('formData.materialIds', (newVal) => {
    // 这个watcher可能还没完全就绪
  })
}

// 此时Playwright填入数据并触发事件
// 但Vue的watcher还没完全激活，事件被忽略或丢失
```

**原因2：Element UI的防抖/节流机制**（可能 15%）
```javascript
// Element UI可能对input事件做了防抖处理
<el-input
  v-model="formData.materialIds"
  @input="debounce(handleInput, 300)"  ← 300ms防抖
/>

// Playwright触发事件后，立即点击搜索
// 但防抖还没执行，数据还没更新
```

**原因3：异步验证未完成**（可能 5%）
```javascript
// Element UI表单可能有异步验证
<el-form-item prop="materialIds" :rules="asyncRules">
  <el-input v-model="formData.materialIds" />
</el-form-item>

// 验证未完成时，formData可能还是旧值
```

---

### 为什么第2次就成功？

```
第1次（失败）：
  打开弹窗 → Vue组件mounted → 初始化响应式数据
  ↓ (填入数据)
  响应式系统还没完全就绪 ❌
  ↓ (关闭弹窗)
  Vue组件没有销毁，保持在内存中（keep-alive?）

第2次（成功）：
  重新打开弹窗 → Vue组件已经就绪（没有重新mounted）
  ↓ (填入数据)
  响应式系统已经激活，立即生效 ✅
  ↓
  搜索成功
```

**关键**：第1次操作虽然失败，但"预热"了Vue的响应式系统


三、可行的解决方案
--------------------------------------------------------------------

### 方案1：等待Vue的nextTick（最优）✅

**原理**：
- Vue.nextTick() 确保在DOM更新完成后执行回调
- 等待Vue的响应式更新周期完成

**实现**：
```javascript
// 在填入素材ID并触发事件后
await idInput.fill(idsText);
await idInput.evaluate(el => {
  el.blur();
  el.dispatchEvent(new Event('change', { bubbles: true }));
  el.dispatchEvent(new Event('input', { bubbles: true }));
});

// 等待Vue的nextTick（关键！）
await page.evaluate(() => {
  return new Promise(resolve => {
    // 尝试访问Vue实例
    const vueInstances = document.querySelectorAll('[data-v-app]');
    if (vueInstances.length > 0 && window.Vue) {
      // 如果能访问到Vue，使用nextTick
      if (window.Vue.nextTick) {
        window.Vue.nextTick(resolve);
      } else {
        // 兼容Vue 3
        setTimeout(resolve, 0);
      }
    } else {
      // 如果访问不到Vue，等待下一个事件循环
      setTimeout(resolve, 0);
    }
  });
});

// 再额外等待一个周期，确保Element UI也完成更新
await page.waitForTimeout(100);
```

**优点**：
- 直接针对根本原因
- 等待时间最短（通常<100ms）
- 理论上100%可靠

**缺点**：
- 依赖Vue实例可访问（可能访问不到）
- 需要测试兼容性

---

### 方案2：第1个账户"预热"弹窗（最稳）✅

**原理**：
- 第1个账户打开弹窗后，先关闭再重新打开
- 第2次打开时，Vue已经就绪

**实现**：
```javascript
async function selectMaterials(page, adGroupDialog, row, materialKeyword, materialIds = [], isFirstAccount = false) {
  // ... 打开素材选择弹窗 ...
  
  // 如果是第1个账户，先"预热"弹窗
  if (isFirstAccount) {
    log(`      第1个账户：预热弹窗状态...`);
    
    // 关闭弹窗
    const closeBtn = materialDialog.locator('.el-dialog__headerbtn .el-dialog__close').first();
    await closeBtn.click({ timeout: 2000 }).catch(() => {});
    await page.waitForTimeout(300);
    
    // 重新打开弹窗
    await row.locator('td').nth(3).click();
    await materialDialog.waitFor({ state: 'visible', timeout: 5000 });
    await page.waitForTimeout(300);
    
    log(`      ✓ 弹窗预热完成`);
  }
  
  // 继续正常的素材选择流程...
  await changePageSize(page, materialDialog, '30');
  // ...
}
```

**优点**：
- 100%可靠（模拟第2次打开的成功状态）
- 不依赖Vue内部机制
- 逻辑清晰易理解

**缺点**：
- 第1个账户多花1-2秒（打开→关闭→重新打开）
- 用户可能看到弹窗闪烁

---

### 方案3：强制更新Vue数据（激进）⚠️

**原理**：
- 直接操作Vue实例，强制更新数据

**实现**：
```javascript
// 填入数据后，直接修改Vue的响应式数据
await page.evaluate((ids) => {
  // 尝试找到Vue实例
  const dialog = document.querySelector('.el-dialog:visible');
  if (dialog && dialog.__vue__) {
    // Vue 2
    const vm = dialog.__vue__;
    if (vm.formData) {
      vm.formData.materialIds = ids;
      vm.$forceUpdate();
    }
  } else if (dialog && dialog.__vueParentComponent) {
    // Vue 3
    const vm = dialog.__vueParentComponent;
    if (vm.ctx.formData) {
      vm.ctx.formData.materialIds = ids;
    }
  }
}, idsText);
```

**优点**：
- 理论上最直接

**缺点**：
- 依赖Vue实例结构（可能访问不到）
- 不同版本Vue结构不同
- 可能破坏Vue的响应式机制
- **不推荐**

---

### 方案4：多次触发事件（折中）✅

**原理**：
- 多次触发input/change/blur事件
- 增加事件被正确捕获的概率

**实现**：
```javascript
// 填入素材ID
await idInput.fill(idsText);
await page.waitForTimeout(100);

// 多次触发事件（尝试3次）
for (let i = 0; i < 3; i++) {
  await idInput.evaluate(el => {
    el.focus();
    el.dispatchEvent(new Event('focus', { bubbles: true }));
  });
  await page.waitForTimeout(50);
  
  await idInput.evaluate(el => {
    el.dispatchEvent(new Event('input', { bubbles: true }));
    el.dispatchEvent(new Event('change', { bubbles: true }));
  });
  await page.waitForTimeout(50);
  
  await idInput.evaluate(el => {
    el.blur();
    el.dispatchEvent(new Event('blur', { bubbles: true }));
  });
  await page.waitForTimeout(100);
}

log(`      ✓ 已多次触发事件，确保Vue捕获`);
```

**优点**：
- 简单直接
- 不依赖Vue内部机制
- 增加成功率

**缺点**：
- 不是100%可靠
- 多花300-500ms

---

### 方案5：点击搜索前再次验证（保守）⚠️

**原理**：
- 点击搜索按钮前，验证input的值是否还存在
- 如果丢失，重新填入

**实现**：
```javascript
// 填入并触发事件
await idInput.fill(idsText);
// ... 触发事件 ...
await page.waitForTimeout(syncWaitTime);

// 验证值是否还在
const currentValue = await idInput.inputValue();
if (currentValue !== idsText) {
  log(`      ⚠️  检测到input值丢失，重新填入`, 'WARN');
  await idInput.fill(idsText);
  await page.waitForTimeout(500);
}

// 点击搜索
await searchBtn.click();
```

**优点**：
- 有额外的安全保障

**缺点**：
- 只检查DOM值，不检查Vue的响应式数据
- 治标不治本


四、推荐方案组合
--------------------------------------------------------------------

### 🎯 最佳组合：方案2（预热） + 方案1（nextTick）

```javascript
async function selectMaterials(page, adGroupDialog, row, materialKeyword, materialIds = [], isFirstAccount = false) {
  // ... 打开素材选择弹窗 ...
  
  // ===== 方案2：第1个账户预热弹窗 =====
  if (isFirstAccount) {
    log(`      第1个账户：预热弹窗状态...`);
    const closeBtn = materialDialog.locator('.el-dialog__headerbtn .el-dialog__close').first();
    await closeBtn.click({ timeout: 2000 }).catch(() => {});
    await page.waitForTimeout(300);
    
    await row.locator('td').nth(3).click();
    await materialDialog.waitFor({ state: 'visible', timeout: 5000 });
    await page.waitForTimeout(300);
    log(`      ✓ 弹窗预热完成`);
  }
  
  // 继续正常流程...
  await changePageSize(page, materialDialog, '30');
  
  // 填入素材ID
  await idInput.fill(idsText);
  await idInput.evaluate(el => {
    el.blur();
    el.dispatchEvent(new Event('change', { bubbles: true }));
    el.dispatchEvent(new Event('input', { bubbles: true }));
  });
  
  // ===== 方案1：等待Vue的nextTick =====
  await page.evaluate(() => {
    return new Promise(resolve => {
      if (window.Vue && window.Vue.nextTick) {
        window.Vue.nextTick(resolve);
      } else {
        setTimeout(resolve, 0);
      }
    });
  });
  await page.waitForTimeout(200); // 其他账户200ms，第1个账户已经预热了
  
  // 点击搜索
  await searchBtn.click();
}
```

**优点**：
- **100%可靠**（方案2保证）
- **速度快**（方案1减少等待时间）
- **双重保障**

**预期效果**：
- 第1个账户：预热1-2秒，后续流程200ms，第1次就成功
- 其他账户：无预热，流程200ms，第1次就成功
