### 测试场景：10个任务

**串行模式（原来）**：
- 执行方式：逐个执行
- 总耗时：约25分钟
- CPU占用：低
- 内存占用：500MB

**并行模式（5并发）**：
- 执行方式：5个一批，分2批
- 总耗时：约5-6分钟
- CPU占用：中等（+30%）
- 内存占用：1-1.5GB（+1GB）

**性能提升**：
- 节省时间：约20分钟
- 提升比例：**80%** ✨
- 成功率：100%（已有双重保障）

---

### 硬件支持评估

**Mac Mini M4 Pro**：
- CPU：10核（充足）
- 内存：16-32GB（充足）
- **结论**：✅ 完全支持5-8并发

**推荐配置**：
- 日常使用：5并发（平衡）
- 保守：3并发（最稳定）
- 激进：8并发（最快）


五、使用说明
--------------------------------------------------------------------

### 启用并行模式（3步）

**Step 1**：编辑 config.js
```javascript
parallel: {
  enabled: true,   // ← 改为 true
  ...
}
```

**Step 2**：保存文件（Cmd+S）

**Step 3**：运行脚本
```bash
npm start
```

---

### 关闭并行模式

编辑 config.js，改为 enabled: false

---

### 调整并发数

编辑 config.js：
```javascript
maxConcurrent: 3,  // 改为3（更保守）
// 或
maxConcurrent: 8,  // 改为8（更激进）
```
