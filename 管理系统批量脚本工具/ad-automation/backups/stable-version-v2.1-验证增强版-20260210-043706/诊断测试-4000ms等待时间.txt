二、测试目的
--------------------------------------------------------------------

**通过极端长的等待时间来确定问题的根本原因：**

### 情况1：4000ms后仍然需要重试 ❌

**结论**：**不是等待时间的问题！**

**可能的真正原因**：
1. **输入框填入的值没有真正生效**
   - DOM的value改变了，但Vue的响应式数据没更新
   - Element UI的表单内部状态没同步

2. **搜索按钮点击时机不对**
   - 点击太早，表单还没验证完成
   - 需要在点击前再等待或触发特定事件

3. **第一次打开弹窗的初始化问题**
   - 某些Vue组件还没完全挂载
   - 事件监听器还没绑定
   - 需要先"激活"一次表单

4. **后端接口的问题**
   - 第一次请求被忽略或超时
   - 需要先发送一次"预热"请求

**下一步方向**：
→ 调查输入框填入逻辑
→ 检查搜索按钮点击时机
→ 考虑在第一个账户前先"预热"弹窗

---

### 情况2：4000ms后第1次就成功 ✅

**结论**：**确实是等待时间不够！**

**说明**：
- Vue/Element UI的数据同步需要非常长的时间
- 可能是第一次打开弹窗时，初始化特别慢

**下一步方向**：
→ 通过二分法找到合适的时间阈值
  - 尝试3000ms、2500ms、2000ms
  - 找到90%成功率的最小时间

→ 或者接受使用4000ms（虽然慢，但稳定）

→ 或者保持自动重试机制（1500ms + 重试，总时间约4-5秒）

### 结果B：4000ms仍需要重试 ❌

**说明**：问题不在等待时间！

**新方向1**：改进输入框填入逻辑
```javascript
// 当前
await idInput.fill(idsText);
await idInput.evaluate(el => {
  el.blur();
  el.dispatchEvent(new Event('change', { bubbles: true }));
});

// 优化：多次触发，确保生效
await idInput.fill(idsText);
await page.waitForTimeout(200);
await idInput.evaluate(el => {
  el.focus();  // 先聚焦
  el.select(); // 选中内容
});
await page.waitForTimeout(200);
await idInput.evaluate(el => {
  el.blur();   // 失焦触发验证
  el.dispatchEvent(new Event('input', { bubbles: true }));
  el.dispatchEvent(new Event('change', { bubbles: true }));
});
```

**新方向2**：等待Vue的nextTick
```javascript
// 触发事件后，等待Vue的更新周期完成
await page.evaluate(() => {
  return new Promise(resolve => {
    if (window.Vue && window.Vue.nextTick) {
      window.Vue.nextTick(resolve);
    } else {
      setTimeout(resolve, 0);
    }
  });
});
```

**新方向3**：先"预热"表单
```javascript
// 第一个账户：先打开弹窗，关闭，再重新打开

```
