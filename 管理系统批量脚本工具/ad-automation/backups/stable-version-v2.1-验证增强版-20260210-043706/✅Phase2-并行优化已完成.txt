二、Phase 2 核心机制
--------------------------------------------------------------------

### 并行处理流程

```
10个任务，5并发：

┌──────────────────────────────────────────────┐
│              第1批（5个任务）                 │
├──────────────────────────────────────────────┤
│ Tab1: 任务1 ┐                                │
│ Tab2: 任务2 ├── 同时执行（2.5分钟）         │
│ Tab3: 任务3 │                                │
│ Tab4: 任务4 │                                │
│ Tab5: 任务5 ┘                                │
└──────────────────────────────────────────────┘
          ↓ 等待2秒
┌──────────────────────────────────────────────┐
│              第2批（5个任务）                 │
├──────────────────────────────────────────────┤
│ Tab1: 任务6  ┐                               │
│ Tab2: 任务7  ├── 同时执行（2.5分钟）        │
│ Tab3: 任务8  │                               │
│ Tab4: 任务9  │                               │
│ Tab5: 任务10 ┘                               │
└──────────────────────────────────────────────┘

总耗时：约5-6分钟（vs 串行25分钟）
性能提升：75-80%
```

---

### 关键特性

1. **多标签页并行**
   - 每个任务独立标签页
   - 互不干扰
   - Promise.all() 等待所有完成

2. **批次控制**
   - 自动分批（5个/批）
   - 批次间延迟（2秒）
   - 避免服务器压力

3. **错误隔离**
   - 单个任务失败不影响其他
   - 独立 try-catch
   - 详细错误报告

4. **资源管理**
   - 任务完成后自动关闭标签页
   - 释放内存
   - 避免资源泄漏

5. **灵活切换**
   - config.parallel.enabled 控制
   - 串行模式作为后备
   - 无需修改代码

六、故障排查
--------------------------------------------------------------------

### 问题1：内存占用过高

**现象**：
- 浏览器卡顿
- 系统响应慢
- 活动监视器显示内存压力

**解决方案**：
```javascript
parallel: {
  enabled: true,
  maxConcurrent: 3,  // 降低并发数
  batchDelay: 3000,  // 增加批次间延迟
}
```

---

### 问题2：某个任务卡住

**现象**：
- 其他任务完成，一个任务一直等待
- 日志显示"等待第X批完成"

**原因**：
- 单个任务遇到问题（弹窗、超时等）
- Promise.all() 等待所有任务

**解决方案**：
- 任务有10分钟超时保护
- 超时后自动失败，不会永远卡住
- 如果经常超时，考虑降低并发数

---

### 问题3：服务器限流

**现象**：
- 多个任务同时失败
- 提示"请求过快"或"服务器忙"

**解决方案**：
```javascript
parallel: {
  enabled: true,
  maxConcurrent: 3,     // 降低并发数
  batchDelay: 5000,     // 增加批次间延迟到5秒
}
```

---

### 问题4：某些任务失败

**现象**：
- 并行时失败，串行时成功
- 失败的是第2、3个任务

**原因**：
- 第1个账户搜索问题（已通过激活输入框解决）
- 资源竞争

**解决方案**：
- 已实施激活输入框机制
- 如仍有问题，降低并发数

八、技术细节
--------------------------------------------------------------------

### Promise.all() 并行机制

```javascript
// lib/parallel.js 核心代码
const promises = batch.map(async (task, index) => {
  const page = await browser.newPage();  // 独立标签页
  try {
    await executeTaskGroup(page, task, ...);
    return { success: true };
  } catch (error) {
    return { success: false, error };
  } finally {
    await page.close();  // 确保关闭
  }
});

// 等待所有任务完成
const results = await Promise.all(promises);
```

**特性**：
- 所有任务同时启动
- 等待全部完成才返回
- 单个失败不影响其他
- 自动资源清理

---

### 错误隔离机制

```javascript
// 每个任务独立try-catch
try {
  await executeTaskGroup(...);
  return { success: true };
} catch (error) {
  // 记录错误，但不抛出
  return { success: false, error: error.message };
}

// Promise.all确保等待所有任务
await Promise.all(promises);  // 即使有失败也会等待完成
```

---

### 内存管理

```javascript
// 任务完成后立即关闭标签页
try {
  await executeTaskGroup(...);
} finally {
  await page.close();  // 在finally中确保关闭
}
```

**效果**：
- 及时释放内存
- 避免内存泄漏
- 支持长时间运行
