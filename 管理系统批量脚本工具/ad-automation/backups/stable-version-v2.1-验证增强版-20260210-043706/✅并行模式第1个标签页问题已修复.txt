问题：第1个标签页不动，只有后4个标签页运行

一、问题描述
--------------------------------------------------------------------

**用户反馈**：
"一共5个标签页，第一个完全不动，后面四个是正常运行的"

**问题现象**：
- 启用并行模式（5并发）
- 看到5个Chrome标签页打开
- 第1个标签页：停留在批量工具页面，不执行任务 ❌
- 第2-5个标签页：正常并行执行任务 ✅


二、根本原因
--------------------------------------------------------------------

### 原有逻辑

```javascript
// index.js
Step 1: 启动浏览器 → 创建 context 和 page
Step 2: 调用 preparePage(page) → 打开第1个标签页
        - 用户手动登录
        - 用户按Enter确认准备就绪
Step 3: 执行任务
        - 串行模式：使用 preparePage 创建的 page
        - 并行模式：使用 context 创建新标签页 ← 问题在这！
```

**问题根源**：
- preparePage 创建了第1个标签页
- 用户在这个标签页中登录并准备
- **但并行模式创建了新的标签页（2-5），没有使用第1个**
- 所以第1个标签页一直停留，没有执行任务

---

### 为什么会这样？

```javascript
// 串行模式（正常）
Step 2: preparePage(page) → 第1个标签页
Step 3: for循环
          - 第1个任务：使用 page（不刷新）
          - 第2个任务：刷新 page
          - 第3个任务：刷新 page
          ...

// 并行模式（有问题）
Step 2: preparePage(page) → 第1个标签页（用于登录）
Step 3: runParallelMode(context, tasks)
          - 创建第2个标签页 → 执行任务1
          - 创建第3个标签页 → 执行任务2
          - 创建第4个标签页 → 执行任务3
          - 创建第5个标签页 → 执行任务4
          ❌ 第1个标签页没有被使用！
```


三、解决方案
--------------------------------------------------------------------

### 修改逻辑

```javascript
// index.js 新逻辑
Step 2: preparePage(page) → 第1个标签页
        - 用户手动登录（persistentContext共享登录状态）
        - 用户按Enter确认
        
Step 3: if (并行模式) {
          关闭第1个标签页 ← 🔥 关键修复！
          // 第1个标签页的使命完成（登录），关闭它
        }
        
Step 4: 执行任务
        - 串行模式：使用第1个标签页（不关闭）
        - 并行模式：创建新的5个标签页，同时执行
```

---

### 修改内容

**文件1**: `index.js`（第54-69行）

```javascript
// 修改前
if (!config.parallel.enabled) {
  await preparePage(page);  // 只有串行模式调用
} else {
  log('并行模式：跳过页面准备');  // 跳过准备 ❌
}

// 修改后
await preparePage(page);  // 所有模式都调用（用于登录）

if (config.parallel.enabled) {
  await page.close();  // 🔥 关键：并行模式关闭准备页面
  log('已关闭准备页面，将在新标签页中并行执行');
}
```

**文件2**: `lib/parallel.js`（第57行）

```javascript
// 修改前
const isFirst = (batchIndex === 1 && index === 0);
await executeTaskGroup(page, taskGroup, ..., isFirst);
// 问题：第2-5个任务因为isFirst=false会重复刷新

// 修改后
await executeTaskGroup(page, taskGroup, ..., true);
// 所有任务都传true，因为都是新标签页，不需要刷新
```

五、技术细节
--------------------------------------------------------------------

### 为什么要关闭准备页面？

**原因1：避免混淆**
- preparePage的页面是用于登录和准备的
- 并行执行时，每个任务都在独立的标签页中
- 保留准备页面会让用户困惑（为什么这个页面不动？）

**原因2：节省资源**
- 准备页面已完成使命（登录）
- 关闭它可以释放一个标签页的内存
- 5个标签页 vs 6个标签页（减少20%资源占用）

**原因3：清晰的逻辑**
- 准备阶段：1个标签页（登录）
- 执行阶段：N个标签页（并行执行）
- 职责分明

---

### persistentContext的作用

```javascript
// launchBrowser()
const context = await chromium.launchPersistentContext(
  profileDir,  // 持久化用户数据目录
  { ... }
);
```

**关键特性**：
- 所有标签页共享登录状态
- 用户在第1个标签页登录后，所有新标签页自动登录 ✅
- 这就是为什么可以先登录，然后关闭第1个标签页的原因

---

### 为什么所有任务都传isFirst=true？

**executeTaskGroup的isFirst参数**：
```javascript
if (!isFirst) {
  await page.goto(taskUrl);  // 刷新页面
}
```

**在串行模式下**：
- 第1个任务：isFirst=true，不刷新（用户准备好了）
- 其他任务：isFirst=false，刷新（清除上一个任务状态）

**在并行模式下**：
- 每个任务都在新标签页中
- parallel.js已经调用了page.goto()
- 所以都传isFirst=true，避免重复刷新
- **节省时间**：每个任务省去一次刷新（约1-2秒）
