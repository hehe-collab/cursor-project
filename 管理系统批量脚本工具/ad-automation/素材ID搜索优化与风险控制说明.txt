一、素材ID搜索问题修复
--------------------------------------------------------------------

【问题现象】
偶尔出现素材ID搜索返回错误数量的情况，例如：
- 输入7个素材ID，但搜索结果显示30个素材
- 这表示素材ID没有正确填入或搜索失败，系统返回了默认的全部素材列表

【根本原因】
1. 素材ID输入框填入后，可能存在残留内容干扰
2. 填入后没有正确触发输入框的 blur 事件
3. 缺少对搜索结果的严格验证

【解决方案】
1. 填入前先清空输入框（避免残留内容）
2. 填入后触发 blur 事件，确保内容生效
3. 添加严格的结果数量验证：
   - 如果搜索结果数量与输入的素材ID数量偏差超过20%
   - 系统会暂停并提示手动检查
   - 防止选择错误的素材

【修改位置】
lib/automation.js - selectMaterials() 函数（约第854-915行）

【验证逻辑】
- 输入7个ID，搜索结果应该是5-9个（允许±20%误差）
- 如果搜索结果是30个，明显异常，会暂停并提示
- 如果搜索结果是6个或8个，视为正常（可能有ID不存在）


二、风险控制功能
--------------------------------------------------------------------

为了防止误操作导致账户预算过高或出价不合理，增加了自动风险控制。

【出价限制】

1. 优化目标为"价值"时：
   - 最低出价：1.1（低于此值自动调整为1.1）
   - 最高出价：无上限
   
2. 优化目标为"转化"时：
   - 最低出价：无下限
   - 最高出价：1.3（超过此值会暂停，提示手动修改）

【预算限制】

- 最低预算：50（低于此值自动调整为50）
- 最高预算：5000（超过此值会暂停，提示手动修改）

【触发机制】

1. 自动调整（不暂停）：
   - 预算 < 50 → 自动调整为 50
   - 优化目标为"价值"且出价 < 1.1 → 自动调整为 1.1
   
2. 强制暂停（需手动确认）：
   - 预算 > 5000 → 暂停并提示
   - 优化目标为"转化"且出价 > 1.3 → 暂停并提示

【日志示例】

正常情况：

  [23:45:56] ✓ 风险控制检查通过: 出价=1.8, 预算=300, 目标=价值
  [23:45:57] ✅ 出价: 1.8, 预算: 300

自动调整：
  [23:45:56] 📋 输入出价: 0.8, 预算: 30
  [23:45:56] ⚠️  优化目标为"价值"时，出价 0.8 低于最低限制 1.1，已调整为 1.1
  [23:45:56] ⚠️  预算 30 低于最低限制 50，已调整为 50
  [23:45:56] ✓ 风险控制检查通过: 出价=1.1, 预算=50, 目标=价值

强制暂停：
  [23:45:56] 📋 输入出价: 1.5, 预算: 6000

  (脚本暂停，等待用户确认)

【修改位置】
lib/automation.js - inputBidAndBudget() 函数（约第1117-1210行）


三、使用建议
--------------------------------------------------------------------

1. Excel数据填写：
   - 确保出价和预算在合理范围内
   - 价值目标：出价 ≥ 1.1
   - 转化目标：出价 ≤ 1.3
   - 预算：50 ≤ 预算 ≤ 5000

2. 素材ID填写：
   - 每个ID一行，或用分号分隔
   - 不要有多余的空格或特殊字符
   - 建议每次不超过30个ID（单页限制）

3. 任务监控：
   - 脚本运行时注意观察日志
   - 如果出现"⚠️"警告，说明有自动调整
   - 如果出现"❌"错误并暂停，需要手动检查并确认

4. 故障处理：
   - 如果素材ID搜索结果数量异常，会自动暂停
   - 此时可以手动在浏览器中重新搜索素材
   - 确认素材正确后，按Enter继续

  await pauseForUser('素材ID搜索结果数量不匹配...');

【风险控制】
// 预算验证
if (budgetNum < 50) {
  budget = 50; // 自动调整
} else if (budgetNum > 5000) {
  await pauseForUser('预算超过安全上限...'); // 强制暂停
}

// 出价验证（根据优化目标）
if (optimizationTarget === '价值') {
  if (bidNum < 1.1) bid = 1.1; // 自动调整
} else if (optimizationTarget === '转化') {
  if (bidNum > 1.3) {
    await pauseForUser('出价超过安全上限...'); // 强制暂停
  }
}
