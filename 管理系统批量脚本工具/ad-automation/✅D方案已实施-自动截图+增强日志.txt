### 1️⃣ 自动截图功能

**保存位置**：
- 📁 ./screenshots/{任务ID}/
- ✅ 保存在项目目录（外置硬盘），不占用系统盘！

**截图时机**（每个任务5张截图）：
1. `01-主体和账户选择.png` - 主体和所有账户选择完成后
2. `02-项目设置.png` - 推广项目设置完成后
3. `03-广告组设置.png` - 所有广告组设置完成后
4. `04-提交前最终确认.png` - 点击提交按钮前
5. `05-提交成功.png` - 第一次提交成功后

**截图特点**：
- ✅ 全页面截图（fullPage: true）
- ✅ 质量优化（默认80%，节省空间）
- ✅ 时间戳命名，方便追溯
- ✅ 按任务ID分文件夹，清晰易查

**示例文件名**：
```
screenshots/
├── 001/
│   ├── 01-主体和账户选择_2026-02-10T15-30-45.png
│   ├── 02-项目设置_2026-02-10T15-31-10.png
│   ├── 03-广告组设置_2026-02-10T15-32-05.png
│   ├── 04-提交前最终确认_2026-02-10T15-32-30.png
│   └── 05-提交成功_2026-02-10T15-32-35.png
└── 002/
    ├── ...
```

---

### 2️⃣ 增强验证日志

**功能**：
在关键操作后，实时显示 Excel配置 vs 实际执行值对比。

**验证时机**：
- ✅ 选择每个账户后 → 验证账户ID
- ✅ 输入出价后 → 验证出价金额
- ✅ 输入预算后 → 验证预算金额

**日志格式**：
```
  📋 Excel配置: 账户1 = 123456789
  ✓ 实际执行: 123456789
  🔍 是否一致: ✅

  📋 Excel配置: 出价 = 1.5
  ✓ 实际执行: 1.5
  🔍 是否一致: ✅

  📋 Excel配置: 预算 = 100
  ✓ 实际执行: 100
  🔍 是否一致: ✅
```

**不一致时的警告**：
```
  📋 Excel配置: 出价 = 1.5
  ✓ 实际执行: 1.2
  🔍 是否一致: ❌
  ⚠️  数据不一致！字段"出价"，Excel为"1.5"，实际为"1.2"
```


二、配置说明
--------------------------------------------------------------------

### config.js 新增配置项

```javascript
// ===== 验证配置 =====
validation: {
  enableScreenshot: true,       // 是否启用自动截图
  enableValidationLog: true,    // 是否启用增强验证日志
  screenshotDir: './screenshots', // 截图保存目录
  screenshotQuality: 80,          // 截图质量（0-100）
},
```

**如何关闭截图/日志**：
```javascript
// 关闭截图（节省空间）
enableScreenshot: false,

// 关闭验证日志（简化输出）
enableValidationLog: false,
```


三、使用方法
--------------------------------------------------------------------

### 1. 正常使用（无需额外操作）

```bash
npm start
```

脚本会自动在关键步骤截图和验证，您无需任何额外操作！

---

### 2. 执行后检查

**步骤1：查看终端日志**
- 观察 🔍 验证日志，确认是否有 ❌ 不一致警告
- 如果全是 ✅，说明数据完全一致

**步骤2：查看截图**
```bash
# 打开截图文件夹
open ./screenshots/
```

- 每个任务ID有独立文件夹
- 按时间顺序查看5张截图
- 逐张对比是否与Excel配置一致

**步骤3：手动抽查（推荐）**
- 随机选择几个任务
- 登录后台系统
- 对比广告配置与Excel是否一致


四、空间占用估算
--------------------------------------------------------------------

**单张截图大小**：
- 全页面截图：约 200-500 KB（质量80%）
- 5张截图/任务：约 1-2.5 MB/任务

**100个任务**：
- 截图总大小：约 100-250 MB
- 保存位置：外置硬盘（不占用系统盘）

**定期清理**：
```bash
# 删除7天前的截图
find ./screenshots -type f -mtime +7 -delete

# 或者手动删除不需要的任务文件夹
rm -rf ./screenshots/001/
```


五、技术实现
--------------------------------------------------------------------

### 新增函数

**1. takeScreenshot(page, taskId, stepName, accountId)**
- 自动创建任务文件夹
- 生成带时间戳的文件名
- 全页面截图
- 失败不影响主流程

**2. logValidation(field, expected, actual)**
- 对比Excel配置与实际值
- 一致显示 ✅，不一致显示 ❌
- 不一致时额外输出警告

**3. validateInputValue(input, field, expected)**
- 读取输入框实际值
- 调用 logValidation 进行对比
- 失败不影响主流程

七、常见问题
--------------------------------------------------------------------

### Q1: 截图占用空间太大怎么办？

**方案1：降低截图质量**
```javascript
// config.js
screenshotQuality: 60,  // 降低到60%
```

**方案2：关闭截图**
```javascript
// config.js
enableScreenshot: false,
```

**方案3：定期清理**
```bash
# 只保留最近3天的截图
find ./screenshots -type f -mtime +3 -delete
```

---

### Q2: 如何只保留出错任务的截图？

当前版本会为所有任务截图。如果您只想保留出错任务的截图，
可以在每批任务完成后，手动删除成功任务的截图文件夹。

未来版本可以考虑增加"仅失败时截图"选项。

---

### Q3: 验证日志太多，影响阅读怎么办？

**方案1：关闭验证日志**
```javascript
// config.js
enableValidationLog: false,
```

**方案2：只在不一致时输出**（需要修改代码）
修改 `logValidation` 函数，只在 `!isMatch` 时输出。

---

### Q4: 截图能保存为PDF吗？

当前版本保存为PNG格式。如需PDF，可以使用以下命令：

```bash
# macOS: 将某个任务的截图合并为PDF
cd screenshots/001
convert *.png 任务001.pdf
```

需要安装 ImageMagick：
```bash
brew install imagemagick
```
