## 问题1: 并行模式浏览器空白页 ❌

### 原因分析
- 并行模式下跳过了初始页面加载
- 新标签页创建时没有登录状态上下文
- `launchPersistentContext` 需要初始页面加载来建立会话

### 解决方案 ✅
```javascript
// 无论串行还是并行，都先加载初始页面
await navigateToMaterialPage(page);
await pauseForUser('请确认已登录...');

// 然后再创建新标签页执行任务
// 新标签页会继承登录状态
```

### 修复文件
- `index.js`: 移除了并行模式跳过初始页面的逻辑

---

## 问题2: 任务ID简化 ✏️

### 需求
- 支持纯数字任务ID（1、2、3...）
- 不再强制使用"任务1"、"任务2"格式

### 解决方案 ✅
```javascript
// Excel读取时，显式转换为字符串
taskId: row['任务ID'] ? String(row['任务ID']).trim() : `任务${index + 1}`
```

### 支持格式
| Excel中填写 | 识别结果 |
|------------|---------|
| 1          | "1"     |
| 2          | "2"     |
| 任务1       | "任务1"  |
| My Task    | "My Task" |
| 空值       | "任务1"（自动生成） |

### 修复文件
- `lib/excel.js`: 强制转换为字符串
- `create-template.js`: 示例数据改为纯数字 1、2、3
