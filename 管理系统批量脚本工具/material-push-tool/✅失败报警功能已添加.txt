## ✨ 新增功能

### 1️⃣ 终端铃声报警（最简单方案）

**触发时机**：
- ✅ 任务执行失败时：立即发出铃声
- ✅ 所有任务完成时：如有失败，再次提醒

**声音效果**：
- 🔔 系统默认提示音（beep一声）
- ⚡ 即时响应，无需等待
- 🎯 简单有效，不需要额外依赖

**实现方式**：
```javascript
process.stdout.write('\x07'); // 终端铃声
```

## 📋 修改的文件

### 1. `lib/parallel.js`（并行模式）

**修改内容**：
- ✅ 添加 `failedTasks` 数组收集失败任务
- ✅ 任务失败时立即发出铃声：`process.stdout.write('\x07')`
- ✅ 保存失败任务的详细信息（任务ID、错误原因、任务标签）
- ✅ 每批次结束显示失败任务列表
- ✅ 返回 `failedTasks` 给调用者

**关键代码**：
```javascript
// 任务失败时
catch (err) {
  // 🔔 发出铃声
  process.stdout.write('\x07');
  
  log(`❌ ${taskLabel} - 失败: ${err.message}`, 'ERROR');
  return { 
    success: false, 
    taskId: task.taskId, 
    error: err.message,
    taskLabel: taskLabel 
  };
}

// 批次结束显示失败任务
if (batchFailedTasks.length > 0) {
  log(`\n   ⚠️  本批失败任务详情：`, 'ERROR');
  batchFailedTasks.forEach((task, idx) => {
    log(`      ${idx + 1}. ${task.taskLabel}`, 'ERROR');
    log(`         原因：${task.error}`, 'ERROR');
  });
}
```

---

### 2. `index.js`（主入口）

**修改内容**：
- ✅ 添加 `failedTasks` 数组接收并行/串行模式的失败任务
- ✅ 串行模式失败时也发出铃声
- ✅ 总结时显示完整的失败任务列表
- ✅ 有失败时总结再次发出铃声提醒

**关键代码**：
```javascript
// 串行模式失败时
catch (err) {
  // 🔔 发出铃声
  process.stdout.write('\x07');
  
  log(`❌ 任务 ${task.taskId} 执行出错: ${err.message}`, 'ERROR');
  failCount++;
  failedTasks.push({
    taskId: task.taskId,
    error: err.message,
    taskLabel: `任务 ${task.taskId}`
  });
  // ...
}

// 总结时
if (failCount > 0) {
  // 🔔 再次提醒
  process.stdout.write('\x07');
  
  log(`❌ 失败: ${failCount} 个`, 'ERROR');
  
  // 显示失败任务详情
  if (failedTasks.length > 0) {
    log('║                  ⚠️  失败任务详情                          ║', 'ERROR');
    failedTasks.forEach((task, idx) => {
      log(`  ${idx + 1}. ${task.taskLabel || task.taskId}`, 'ERROR');
      log(`     失败原因：${task.error}`, 'ERROR');
    });
  }
}
```

## 🎯 使用建议

### 1. 铃声提醒

**什么时候响**：
- ✅ 每个任务失败时立即响
- ✅ 所有任务完成后，如有失败再响一次

**适用场景**：
- 🎧 戴着耳机工作时：能第一时间听到
- 💼 离开电脑时：走回来能听到总结提醒
- 🖥️ 多窗口工作时：即使没看终端也能知道

### 2. 失败列表查看

**批次级别**：
- 每批次完成后立即看到本批的失败任务
- 适合快速定位问题

**总结级别**：
- 所有任务完成后看到完整失败列表
- 适合最终检查和记录

### 3. 故障处理流程

```
听到铃声 🔔
    ↓
查看终端日志
    ↓
找到失败任务详情
    ↓
查看具体错误原因
    ↓
针对性处理问题
    ↓
重新运行该任务（可以单独配置Excel）
```

## ⚙️ 技术细节

### 终端铃声原理

```javascript
process.stdout.write('\x07');
```

- `\x07`：ASCII 控制字符 BEL（Bell）
- 历史：早期电传打字机的铃铛
- 现代：触发终端模拟器的提示音
- 兼容：所有终端（macOS Terminal、iTerm2、VSCode等）

### 数据结构

**失败任务对象**：
```javascript
{
  success: false,           // 失败标记
  taskId: "任务3",          // Excel中的任务ID
  error: "Timeout...",      // 错误详情
  taskLabel: "[批次1-任务3] 任务3"  // 完整标签
}
```
