🚀 工具1：TikTok广告自动化工具
═══════════════════════════════════════════════════════════

### Step 1：进入目录
```bash
cd /Volumes/存钱罐/cursor项目/ad-automation
```

### Step 2：生成Excel模板（仅首次）
```bash
npm run template
```
→ 生成 `tasks.xlsx`

### Step 3：填写广告数据
打开 `tasks.xlsx`，填写：
- 任务ID：1, 2, 3...
- 广告系列ID：从后台复制
- 账户ID：从后台复制
- 广告名称：自定义
- 投放目标：价值 或 转化
- 出价：注意风控限制
- 预算：50-5000
- 素材ID：多个用逗号分隔（优先）
- 素材关键词：如果素材ID为空

### Step 4：启动工具
```bash
npm start
```

### Step 5：操作流程
1. ✅ 查看任务清单和风控验证
2. ✅ 浏览器自动打开
3. ⚠️ **首次需要手动登录**（登录状态会保存）
4. ✅ 按 **Enter** 开始执行
5. ✅ 观察10个标签页同时创建广告
6. ✅ 完成后查看统计结果
7. ✅ 按 **Enter** 关闭浏览器

### ⏱️ 预计耗时
- 10个任务：~3分钟
- 30个任务：~9分钟

🎨 工具2：素材推送自动化工具
═══════════════════════════════════════════════════════════

### Step 1：进入目录
```bash
cd /Volumes/存钱罐/cursor项目/material-push-tool
```

### Step 2：生成Excel模板（仅首次）
```bash
npm run template
```
→ 生成 `data/push-tasks.xlsx`

### Step 3：填写推送数据
打开 `data/push-tasks.xlsx`，填写：
- 任务ID：1, 2, 3...（可选）
- 源账户ID：素材来源账户
- 素材关键词：搜索关键词
- 目标账户ID列表：每行一个账户ID
- 备注：可选

**示例**：
```
任务ID: 1
源账户ID: C-MADxd-IDN-2-8W-11
素材关键词: 桶礼盒波
目标账户ID列表:
  7561360888110
  7561360897493
  7561360323361
```

### Step 4：启动工具
```bash
npm start
```

### Step 5：操作流程
1. ✅ 查看任务清单
2. ✅ 浏览器自动打开素材页面
3. ⚠️ **确认已登录**（与广告工具共享登录状态）
4. ✅ 按 **Enter** 开始执行
5. ✅ 观察10个标签页同时推送素材
6. ✅ 完成后查看统计结果
7. ✅ 按 **Enter** 关闭浏览器

### ⏱️ 预计耗时
- 6个任务（~100素材）：~2分钟
- 18个任务（~300素材）：~4分钟

⚙️ 调整并发数
═══════════════════════════════════════════════════════════

如果觉得10个并发太快或太慢，可以调整：

### 修改 config.js
```javascript
parallel: {
  enabled: true,
  maxConcurrent: 5,    // 改为 3-10 之间的数字
  batchDelay: 2000,
}
```

**建议**：
- M4 Pro：10 ⭐
- M1/M2：5-8
- 保守值：3-5

💡 使用技巧
═══════════════════════════════════════════════════════════

### 技巧1：先测试再批量
```bash
# 先用 npm test 验证Excel数据
npm test

# 确认无误后再启动
npm start
```

### 技巧2：分批执行大任务
如果有100个任务：
- 第1批：任务1-30
- 第2批：任务31-60
- 第3批：任务61-100

避免一次性执行过多，便于监控。

### 技巧3：关注终端日志
- ✅ 绿色 [OK]：成功
- ⚠️ 黄色 [WARN]：警告（通常无害）
- ❌ 红色 [ERROR]：错误（会自动重试）

### 技巧4：利用截图验证（广告工具）
启用截图功能：
```javascript
// config.js
screenshot: {
  enabled: true,
}
```
截图保存在：`/Volumes/存钱罐/ad-screenshots/`

🛠️ 故障排除
═══════════════════════════════════════════════════════════

### 问题1：浏览器无法启动
**解决**：
```bash
# 检查浏览器是否存在
ls pw-browsers/

# 如果不存在，重新安装
npx playwright install chromium
```

### 问题2：Excel数据格式错误
**解决**：
```bash
# 使用测试命令检查
npm test

# 根据提示修正Excel数据
```

### 问题3：某个步骤总是失败
**解决**：
- 检查网络连接
- 确认后台页面未改版
- 查看终端错误日志
- 手动执行一次该步骤，观察问题

### 问题4：登录状态丢失
**解决**：
```bash
# 删除旧的登录数据
rm -rf browser-data/

# 重新启动，手动登录
npm start
```

### 问题5：并发太高，电脑卡顿
**解决**：
```javascript
// 降低并发数
parallel: {
  maxConcurrent: 3,   // 降低到3
}
```
