═══════════════════════════════════════════════════════════

本项目包含两个完整的自动化工具，用于TikTok广告投放和素材管理：

## 工具1：TikTok广告自动化工具 (ad-automation)
**功能**：批量创建TikTok广告
**版本**：v2.0 (并行模式)
**路径**：`/Volumes/存钱罐/cursor项目/ad-automation/`

## 工具2：素材推送自动化工具 (material-push-tool)
**功能**：跨账户批量推送素材
**版本**：v1.0
**路径**：`/Volumes/存钱罐/cursor项目/material-push-tool/`

⚙️ 并行配置
═══════════════════════════════════════════════════════════

两个工具都支持并行执行，配置方式相同：

### 启用并行
```javascript
// config.js
parallel: {
  enabled: true,      // 设为 true
  maxConcurrent: 10,  // 并发数：3-10
  batchDelay: 2000,   // 批次间隔
}
```

### 禁用并行（回退串行）
```javascript
parallel: {
  enabled: false,     // 设为 false
}
```

### 推荐配置
- **M4 Pro**：10并发
- **M1/M2**：5-8并发
- **Intel Mac**：3-5并发

## 典型工作流

### 场景1：新素材投放
```
1. 使用"素材推送工具"将素材推送到所有账户
   → 10个账户，3分钟完成
   
2. 使用"广告自动化工具"批量创建广告
   → 30个广告（10账户×3广告），5分钟完成
```

### 场景2：快速测试素材
```
1. 使用"素材推送工具"推送到测试账户
   → 1分钟完成
   
2. 使用"广告自动化工具"创建测试广告
   → 小预算快速测试
```

### 场景3：批量复制广告
```
使用"广告自动化工具"：
- 相同素材ID
- 不同账户
- 快速复制成功广告的配置
```

🔧 常见问题
═══════════════════════════════════════════════════════════

### Q1：浏览器一直下载怎么办？
**A**：两个工具共享浏览器，只需下载一次
- 第一次运行会自动下载（~300MB）
- 后续运行直接使用
- 位置：`pw-browsers/`

### Q2：并发数设置多少合适？
**A**：根据机器性能：
- M4 Pro：10（推荐）
- M1/M2：5-8
- 保守值：3-5

### Q3：任务失败了怎么办？
**A**：自动重试机制：
- 每个步骤最多重试3次
- 重试失败会暂停等待用户干预
- 手动处理后按 Enter 继续

### Q4：如何验证数据准确性？
**A**：
- 广告工具：启用截图功能
- 素材工具：手动检查后台素材库
- 两个工具都会显示详细日志

### Q5：出现"策略1失败"警告？
**A**：已优化，重新运行不会再出现

### Q6：第一个账户总是需要重试？
**A**：已解决（预热机制 + 激活机制）

🛡️ 风险控制
═══════════════════════════════════════════════════════════

## 广告自动化工具

### 出价控制
- **价值目标**：≥ 1.1（防止出价过低）
- **转化目标**：≤ 1.3（防止出价过高）

### 预算控制
- **最低**：50（防止预算过小）
- **最高**：5000（防止账户"跑炸"）

### 验证时机
1. **Excel读取时**：预验证，提前发现问题
2. **提交前**：二次验证，最后防线

### 超限处理
```
⚠️ 检测到风险：
  账户ID: 7560621...
  出价: 2.0（超过1.3限制）
  → 自动拦截，不会提交
  → 暂停等待用户修正
```
