## 🔍 问题诊断

### 用户反馈的问题

1. **素材推送不完整**
   - 源账户30个素材，只推送了20个
   - 源账户24个素材，只推送了18个

2. **观察到的异常现象**
   - 翻页还没成功，就点击了全选按钮
   - 本页素材还没推送，就直接翻页了

### 根本原因分析

**原来的代码**（第268行）：
```javascript
async function goToNextPage(page) {
  const nextPageBtn = page.locator('button.btn-next').first();
  await nextPageBtn.click();
  
  // ❌ 问题：翻页后只固定等待500ms
  await page.waitForTimeout(config.delay.beforeNextPage);  // 500ms
  
  log(`✅ 已翻页`, 'OK');
}
```

**问题**：
- ❌ 固定等待500ms，**页面可能还没加载完**
- ❌ 表格数据可能还在更新中
- ❌ DOM可能还在渲染
- ❌ 就立即执行下一步（全选素材）

**导致的后果**：
```
翻页点击 → 等待500ms → 全选 ❌（表格还是上一页的数据！）
                    → 推送 ❌（推送了错误的素材或空数据）
                    → 检查下一页 → 翻页
```

**尤其在并行模式下**：
- 10个标签页同时执行
- CPU、内存、网络高负载
- 页面渲染变慢
- **500ms根本不够！**

## ✅ 解决方案：方案A - 等待内容真正改变

### 核心思路

**不依赖固定时间**，而是**等待页面内容实际改变**。

### 实现逻辑

```javascript
async function goToNextPage(page) {
  // 1. 翻页前：记录当前第一个素材ID
  const oldFirstMaterialId = await page.locator('tbody tr').first()
    .locator('td').nth(1).textContent()
    .catch(() => '');
  
  // 2. 点击下一页
  await nextPageBtn.click();
  
  // 3. ⭐ 等待表格内容真正改变
  await page.waitForFunction((oldId) => {
    const newFirstRow = document.querySelector('tbody tr');
    if (!newFirstRow) return false;  // 表格还没加载
    
    const newFirstId = newFirstRow.querySelector('td:nth-child(2)')?.textContent || '';
    
    // 确保：1) 内容变了  2) 不是空的
    return newFirstId !== oldId && newFirstId.trim() !== '';
  }, oldFirstMaterialId, { timeout: 10000 });
  
  // 4. 再等800ms确保DOM完全渲染
  await page.waitForTimeout(800);
}
```

### 关键改进

| 对比项 | 原来（固定等待） | 现在（智能等待） |
|--------|----------------|----------------|
| **等待方式** | 固定500ms | 等待内容改变（最多10秒） |
| **可靠性** | 低（可能不够） | 高（确保真正改变） |
| **页面快时** | 浪费时间 | 快速通过 |
| **页面慢时** | ❌ 不够（导致错误） | ✅ 等够（直到改变） |
| **并行模式** | ❌ 经常出错 | ✅ 稳定可靠 |

```javascript
async function goToNextPage(page) {
  log(`翻到下一页`, 'STEP');
  
  // 1. 记录翻页前的第一个素材ID
  const oldFirstMaterialId = await page.locator('tbody tr').first()
    .locator('td').nth(1).textContent()
    .catch(() => '');
  
  log(`  翻页前第一个素材ID: ${oldFirstMaterialId.substring(0, 20)}...`);
  
  // 2. 点击下一页
  const nextPageBtn = page.locator('button.btn-next').first();
  await nextPageBtn.click();
  
  // 3. ⭐ 等待页面内容真正改变
  try {
    await page.waitForFunction((oldId) => {
      const newFirstRow = document.querySelector('tbody tr');
      if (!newFirstRow) return false;
      
      const newFirstIdCell = newFirstRow.querySelector('td:nth-child(2)');
      if (!newFirstIdCell) return false;
      
      const newFirstId = newFirstIdCell.textContent || '';
      return newFirstId !== oldId && newFirstId.trim() !== '';
    }, oldFirstMaterialId, { timeout: 10000 });
    
    const newFirstMaterialId = await page.locator('tbody tr').first()
      .locator('td').nth(1).textContent()
      .catch(() => '');
    
    log(`  翻页后第一个素材ID: ${newFirstMaterialId.substring(0, 20)}...`);
    log(`  ✅ 页面内容已改变`);
    
  } catch (err) {
    log(`  ⚠️ 等待页面改变超时，继续执行`, 'WARN');
  }
  
  // 4. 再等800ms确保DOM渲染
  await page.waitForTimeout(800);
  
  log(`✅ 已翻页`, 'OK');
}
```

## ⚙️ 技术细节

### waitForFunction 的工作原理

```javascript
await page.waitForFunction((oldId) => {
  // 这个函数会在浏览器页面中反复执行
  // 直到返回 true 或超时
  
  const newFirstId = document.querySelector('tbody tr td:nth-child(2)')?.textContent;
  return newFirstId !== oldId && newFirstId.trim() !== '';
}, oldFirstMaterialId, { timeout: 10000 });
```

**执行过程**：
1. 点击翻页后，Playwright 开始执行这个函数
2. 每隔一小段时间（通常几十毫秒）执行一次
3. 检查表格第一行第二列的内容
4. 如果内容还是旧的 → 继续等待
5. 如果内容变了 → 返回 true，结束等待
6. 如果10秒还没变 → 超时，抛出错误（被 catch 捕获）

**优势**：
- ✅ 不浪费时间（页面快就等得少）
- ✅ 足够可靠（页面慢就多等一会）
- ✅ 有上限保护（最多10秒）

### 为什么选择第一个素材ID作为标记？

1. **唯一性**：每个素材的ID都是唯一的
2. **稳定性**：翻页后第一个素材ID必然会变
3. **可见性**：通常在表格的第二列，容易定位
4. **可靠性**：如果ID没变，说明页面肯定没更新
