## 问题：并行标签页不易观察

### 原因
- 标签页创建和任务执行混在一起
- 速度太快，用户看不清楚有几个标签页同时打开

### 解决方案 ✅

**改为两阶段执行**：

#### 🔷 阶段1：批量创建标签页（并行）
```
1. 同时创建所有标签页
2. 同时加载素材页面
3. 等待所有标签页准备完成
→ 用户可以清楚看到 3 个标签页同时出现
```

#### 🚀 阶段2：并行执行任务（并行）
```
1. 所有标签页同时开始执行推送任务
2. 独立完成各自的任务
3. 完成后自动关闭
```

### 浏览器行为
```
1. 按 Enter 后，立即看到浏览器打开 3 个新标签页
2. 3 个标签页同时加载素材页面
3. 加载完成后，同时开始执行推送任务
4. 任务完成后，标签页依次自动关闭
```

## 📊 与之前的区别

| 对比项 | 之前（混合执行） | 现在（分阶段执行） |
|--------|----------------|-------------------|
| 标签页创建 | 逐个创建并执行 | 先全部创建 ⭐ |
| 可观察性 | 难以观察 | 清晰可见 ⭐ |
| 执行速度 | 稍快 | 几乎相同 |
| 日志清晰度 | 混乱 | 分阶段显示 ⭐ |
