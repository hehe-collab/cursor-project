## 1. 功能总览

| 功能模块 | 技术实现 | 输入 | 输出 |
|---------|---------|------|------|
| **调分辨率** | FFmpeg scale + h264_videotoolbox/libx264 | 视频列表 | Scaled_Video/*.mp4 |
| **提取字幕** | macOS Vision OCR + 0.25s 采样 + 多数表决融合 | 视频 + 红框区域 | OCR_Result/*.srt |
| **画面擦除** | FFmpeg delogo 多区域级联 | 视频 + 多红框 | Clean_Video/*.mp4 |
| **AI 翻译** | Ollama API (llama3/deepseek-v3) | SRT 文件 | Translated_SRT/*.srt |
| **字幕压制** | Pillow 逐帧绘制 + FFmpeg 管道 | 视频 + SRT | Final_Thai/*.mp4 |
| **剧集合并** | 亮度检测片尾 + FFmpeg concat | 多集视频文件夹 | merged_output.mp4 |

## 3.5 提取字幕功能优化建议（专项讨论）

### 当前流程概览

1. **采样**：每 0.25 秒取一帧（`cv2.CAP_PROP_POS_FRAMES` + `read`）
2. **裁剪**：按红框区域 + 15px padding 截取 ROI
3. **OCR**：灰度 → 3 倍放大 → 拉普拉斯锐化 → 写临时 jpg → Vision 识别
4. **分组**：`difflib.SequenceMatcher` 相似度 > 0.6 视为同一条字幕
5. **融合**：每组用 `Counter` 取出现最多、最长的文本
6. **时间**：开始 = 组首采样时间，结束 = 组尾时间 + 0.25 秒

### 优化建议（按优先级）

| 类别 | 建议 | 说明 | 风险 |
|-----|------|------|------|
| **采样** | 采样间隔可配置 | 0.25s 对 60 分钟视频约 1.44 万次 OCR，耗时长。可增加 0.2s~0.5s 选项，默认 0.25s | 低 |
| **采样** | 长视频自适应采样 | 如时长 > 10 分钟，自动改为 0.3s 或 0.4s，减少总采样数 | 低 |
| **抽帧** | 考虑 FFmpeg 抽帧 | `cv2` 随机 seek 对部分编码不稳定。可用 `ffmpeg -vf "fps=4"` 等先抽帧再 OCR | 中 |
| **OCR** | 减少临时文件 I/O | 每次识别都写 jpg 再读，可尝试 `VNImageRequestHandler` 的 CVPixelBuffer/CGImage 接口 | 中 |
| **OCR** | 预处理可配置 | 3 倍放大、锐化强度可做成参数，适应不同清晰度 | 低 |
| **分组** | 相似度阈值可调 | 0.6 为经验值，可增加 0.5~0.8 滑块，方便不同语速/风格 | 低 |
| **分组** | 空文本分组逻辑 | 连续空采样会成组，`finalize_group` 会返回 None，但组边界可能异常，可显式跳过空采样 | 低 |
| **时间** | 结束时间更合理 | 固定 +0.25s 可能与实际显示不符，可改为「下一组开始时间 - 0.05s」或类似 | 低 |
| **格式** | SRT 毫秒分隔符 | `format_time` 输出用 `.`，标准 SRT 用 `,`，部分播放器更认逗号 | 极低 |
| **性能** | 结果缓存 | 同视频同区域重复提取时，可缓存 OCR 结果，避免重复计算 | 中 |
| **健壮** | VFR 视频 | 可变帧率视频用 `total_frames/fps` 算时长可能不准，可考虑用 `cv2.CAP_PROP_FRAME_COUNT` 等更稳妥方式 | 中 |

### 不建议轻易改动的部分

- **多数表决逻辑**：`(c[x], len(x))` 排序能较好抑制 OCR 噪点
- **Vision 语言设置**：`zh-Hans` + `en-US` 适合中英混合短剧
- **单字过滤**：`len(clean_text) > 1` 能过滤大量噪点
