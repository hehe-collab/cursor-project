🧭 项目概述
目标：输入一条中文爆款素材 + 中文原剧全集 + 印尼语版全集，自动输出一条剪辑结构完全一致的印尼语版素材。
核心原理：音频指纹匹配定位 + 时间码平移裁切 + 自动拼接。

📋 详细步骤说明
Step 1：为中文原剧建立音频指纹库
输入：chinese_drama/ 下所有剧集 mp4
输出：音频指纹数据库（一部剧建一次，后续复用）

具体逻辑：

遍历 chinese_drama/ 下所有 mp4 文件
用 FFmpeg 提取每集的音轨（转为 wav 格式）
将每集音频注册到指纹库中，文件名带上集数标识（如 ep01、ep02）
指纹库建好后保存，后续同一部剧的所有爆款素材都可以复用

关键点：

每集音频注册时要记录文件名（集数信息），匹配时才能知道命中了哪一集
建库是一次性操作，一部 80 集的短剧大概几分钟就能建完


Step 2：爆款素材场景切割
输入：一条爆款素材 mp4
输出：切割后的片段列表（含每个片段的起止时间码）

具体逻辑：

用 PySceneDetect 的 ContentDetector 检测镜头切换点
输出切割点列表，例如：
[
  {"segment": 1, "start": "00:00:00.000", "end": "00:00:03.200"},
  {"segment": 2, "start": "00:00:03.200", "end": "00:00:07.800"},
  {"segment": 3, "start": "00:00:07.800", "end": "00:00:12.500"}
]


用 FFmpeg 按切割点把素材拆成独立的小视频片段，存到 temp/ 目录
同时提取每个片段的音频（wav），供下一步匹配用

关键参数：

ContentDetector 的 threshold 参数需要调优，短剧素材通常快切较多，建议从 threshold=27 开始试
如果切得太碎（一个镜头被切成多段），调高阈值
如果切得太粗（多个镜头没分开），调低阈值


Step 3：逐片段音频指纹匹配
输入：Step2 切出的每个音频片段 + Step1 建好的指纹库
输出：匹配映射表

具体逻辑：

逐个读取 temp/ 下的音频片段
用音频指纹库进行匹配查询
每个片段返回：命中的集数 + 在该集中的时间偏移量
结合片段时长，计算出在原剧中的起止时间码
输出映射表：

{
  "clip": "clip01.mp4",
  "segments": [
    {
      "segment": 1,
      "clip_start": "00:00:00.000",
      "clip_end": "00:00:03.200",
      "matched_episode": "ep03",
      "episode_start": "00:02:15.300",
      "episode_end": "00:02:18.500",
      "confidence": 0.98
    },
    {
      "segment": 2,
      "clip_start": "00:00:03.200",
      "clip_end": "00:00:07.800",
      "matched_episode": "ep01",
      "episode_start": "00:00:30.500",
      "episode_end": "00:00:35.100",
      "confidence": 0.95
    }
  ]
}

关键点：

设定一个置信度阈值（比如 0.7），低于阈值的标记为匹配失败，进入 Step 5 视觉兜底
如果某个片段特别短（<1秒），音频指纹可能不够，也标记为失败


Step 4：从印尼语版裁切 + 拼接
输入：Step3 的映射表 + indonesian_drama/ 下的剧集
输出：印尼语版素材 mp4

具体逻辑：

读取映射表
对每个 segment，用 FFmpeg 从对应的印尼语版剧集中裁切：
ffmpeg -i indonesian_drama/ep03.mp4 \
       -ss 00:02:15.300 -to 00:02:18.500 \
       -c:v libx264 -c:a aac \
       temp/indo_seg_01.mp4


所有片段裁切完成后，按原素材顺序拼接：
# 生成 filelist.txt
file 'temp/indo_seg_01.mp4'
file 'temp/indo_seg_02.mp4'
file 'temp/indo_seg_03.mp4'
...

# 拼接
ffmpeg -f concat -safe 0 -i filelist.txt -c copy output/clip01_indo.mp4



关键点：

裁切时用 -ss 放在 -i 前面可以加速 seek，但精度可能差几帧；放在 -i 后面精度高但慢一些。建议放在后面，优先保证精度
拼接时如果编码参数不一致可能报错，统一用 -c:v libx264 -c:a aac 重新编码可以避免
最终输出文件的分辨率、帧率应与原素材保持一致


Step 5：视觉匹配兜底（可选）
输入：Step3 中匹配失败的片段
输出：补充的映射关系

具体逻辑：

对匹配失败的片段，提取关键帧（比如取中间帧）
计算关键帧的 pHash
遍历中文原剧所有集数，每隔 0.5 秒取一帧计算 pHash
找到汉明距离最小的帧，定位到对应的集数和时间码
以该帧为锚点，前后扩展片段时长，得到起止时间码

关键点：

这一步计算量较大（需要遍历所有帧），但只处理少量失败片段，可以接受
可以预先为每集建立帧级 pHash 索引，加速查询


⚙️ 配置文件
# config.yaml

# 输入路径
chinese_drama_dir: "./input/chinese_drama"
indonesian_drama_dir: "./input/indonesian_drama"
viral_clips_dir: "./input/viral_clips"

# 输出路径
output_dir: "./output"
temp_dir: "./temp"

# 音频指纹配置
fingerprint_backend: "audfprint"          # "audfprint" 或 "dejavu"
fingerprint_db_path: "./temp/fprint.pklz" # audfprint 指纹库路径

# 场景切割配置
scene_detect_threshold: 27                # ContentDetector 阈值，可调
min_scene_duration: 0.5                   # 最短片段时长（秒），低于此值合并到前一个片段

# 匹配配置
match_confidence_threshold: 0.7           # 音频匹配置信度阈值
enable_visual_fallback: true              # 是否启用视觉兜底

# FFmpeg 配置
video_codec: "libx264"
audio_codec: "aac"


🚀 一键执行流程（pipeline.py）
python src/pipeline.py --clip input/viral_clips/clip01.mp4

执行顺序：
1. 检查指纹库是否存在
   ├── 不存在 → 运行 01_build_fingerprint_db.py 建库
   └── 已存在 → 跳过

2. 运行 02_scene_detect.py
   → 输出片段列表 + 切割后的音频文件

3. 运行 03_match_segments.py
   → 输出映射表 JSON

4. 如果有匹配失败的片段且 enable_visual_fallback=true
   → 运行 05_visual_fallback.py 补充映射

5. 运行 04_cut_and_concat.py
   → 输出最终印尼语版素材到 output/

6. 打印质检报告（总时长对比、各片段匹配情况）


✅ 质检清单
最终输出后自动校验：
| 检查项 | 方法 | 通过标准 |
|-----------|--------|------------|
| 总时长一致性 | 对比原素材与输出素材时长 | 差异 < 0.5秒 |
| 片段数完整性 | 映射表中所有片段都有匹配结果 | 无遗漏 |
| 匹配置信度 | 所有片段置信度 | 均 > 0.7 |
| 画面连续性 | 人工抽检拼接点 | 无明显跳帧/黑帧 |

音频指纹库只需要对中文原剧建一次，因为印尼语版音轨与中文版完全一致，匹配中文版等于匹配印尼语版
时间码可以直接从中文版平移到印尼语版，两个版本画面完全一致，只有烧录字幕不同
audfprint 优先，不依赖数据库，部署简单，先跑通再说
每个 step 独立可运行，方便调试，同时 pipeline.py 串联全流程
映射表用 JSON 存储，方便人工检查和调试
