二、根本原因
--------------------------------------------------------------------

### 搜索功能未就绪

```
打开弹窗
  ↓
【问题】搜索输入框的事件监听器还没完全绑定
  ↓
立即填入搜索条件
  ↓
【问题】Vue/Element UI的响应式数据没有更新
  ↓
点击搜索按钮
  ↓
【问题】后端接收到空参数或未初始化的参数
  ↓
返回默认素材列表（30个或10个）
  ↓
关键词检索：可能选错 ❌
ID检索：验证失败 → 重试 → 成功 ✅
```

---

### 为什么第2次就成功？

```
第1次尝试（失败）：
  打开弹窗 → Vue组件mounted → 初始化中...
  填入搜索条件 → 条件被忽略（事件监听器未就绪）
  点击搜索 → 返回默认列表
  关闭弹窗 → Vue组件保持在内存中

第2次尝试（成功）：
  重新打开弹窗 → Vue组件已就绪（不需要重新mounted）
  填入搜索条件 → 条件立即生效 ✅
  点击搜索 → 返回正确结果
```

**关键**：第1次操作"预热"了搜索功能，第2次就正常了。


三、解决方案：激活搜索输入框
--------------------------------------------------------------------

### 核心思路

**在填入搜索条件前，先"激活"输入框，确保搜索功能完全就绪**

```
打开弹窗
  ↓
🔥 点击输入框（聚焦）
  ↓
🔥 等待200ms
  ↓
🔥 输入一个空格（触发输入框初始化）
  ↓
🔥 等待300ms（让Vue/Element UI完全激活）
  ↓
🔥 清空输入框
  ↓
🔥 等待200ms
  ↓
✅ 此时输入框和搜索功能完全就绪！
  ↓
填入真正的搜索条件
  ↓
等待数据同步（900ms或200ms）
  ↓
点击搜索
  ↓
✅ 第1次就成功！
```

六、技术原理
--------------------------------------------------------------------

### 为什么"输入空格"能激活输入框？

**Element UI的input组件工作机制**：

```javascript
// Element UI内部（简化版）
<el-input v-model="searchValue" @input="handleInput">

// Vue组件
data() {
  return {
    searchValue: ''  // 初始化时可能还没完全绑定
  }
},
mounted() {
  // 绑定事件监听器（需要时间）
  this.$watch('searchValue', (newVal) => {
    // 这个watcher在mounted后才激活
  })
},
methods: {
  handleInput(value) {
    // 这个方法也需要在mounted后才能正常工作
  }
}
```

**激活过程**：
```
1. 打开弹窗 → Vue组件mounted → 初始化响应式数据和事件监听器
   （这需要时间！）

2. Playwright立即操作：
   - 点击输入框 → 触发focus事件 → 确保输入框获得焦点
   - 输入空格 → 触发input事件 → **强制激活Vue的watcher**
   - 等待300ms → 让Vue的响应式更新周期完成
   - 清空 → 确保输入框状态干净
   - 等待200ms → 确保清空操作完成

3. 此时：
   - 输入框已获得焦点 ✅
   - Vue的响应式系统已激活 ✅
   - Element UI的事件监听器已绑定 ✅
   - 搜索功能完全就绪 ✅

4. 填入真正的搜索内容：
   - input.value改变 → 立即触发input事件
   - Vue的watcher立即响应 → 更新searchValue
   - Element UI的handleInput立即执行
   - 点击搜索 → 后端接收到正确参数 ✅
```

---

### 为什么之前的方案不行？

**之前的方案**：
1. 预热弹窗：打开 → 关闭 → 重新打开
   - 问题：复杂，增加1-2秒，治标不治本
2. 延长等待时间：900ms、4000ms
   - 问题：无论等多久，第1次都不生效

**现在的方案**：
- 不是"等待"初始化完成，而是**主动触发**初始化
- 通过输入空格，强制激活所有事件监听器
- 简单高效，不增加太多时间
