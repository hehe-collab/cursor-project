# Node.js + Playwright 项目打包为桌面应用指南

> 本文档归纳 DramaBagus 批量广告工具从 Node.js 脚本打包为 Windows/Mac 可安装程序的完整实现，供其他设备参考学习。

---

## 一、整体架构

### 1.1 技术选型

| 技术 | 用途 |
|------|------|
| **Electron** | 将 Node.js 应用包装为桌面应用，自带 Node 运行时 |
| **electron-builder** | 打包为 Windows .exe / Mac .dmg 安装程序 |
| **Playwright** | 自动化浏览器操作（打包后首次运行自动下载 Chromium） |

### 1.2 运行模式

- **CLI 模式**：`npm start` → `node index.js`，在终端运行，使用 readline 等待用户按 Enter
- **Electron 模式**：`npm run electron` 或运行打包后的 exe/dmg，带图形界面，使用弹窗代替 readline

### 1.3 核心思路

1. 原脚本保持可独立运行（`node index.js`）
2. 新增 Electron 入口，在启动时注入「暂停」「日志」的替代实现
3. 配置文件支持 Electron 路径覆盖（用户数据目录可写）
4. 主逻辑抽取到 `index-electron.js`，供 Electron 调用

---

## 二、新增文件清单

| 文件 | 作用 |
|------|------|
| `electron-main.js` | Electron 主进程：创建窗口、设置路径、IPC、运行自动化 |
| `electron-ui.html` | 界面：日志区、开始按钮、打开目录、暂停弹窗 |
| `electron-preload.js` | 渲染进程与主进程通信桥接 |
| `index-electron.js` | 主逻辑入口，导出 `run()` 供 Electron 调用 |
| `docs/打包实现指南.md` | 本文档 |

## 三、关键实现细节

### 3.1 路径处理（config.js）

**问题**：打包后应用安装在 Program Files 等只读目录，数据需写入用户目录。

**实现**：在 `electron-main.js` 最开头设置 `global.__electronAppPaths`，config.js 检测到后使用 `app.getPath('userData')` 作为根路径：

```javascript
// electron-main.js 最开头（任何 require config 之前）
const appData = app.getPath('userData');
global.__electronAppPaths = { appData, exeDir, resourcesDir };

// config.js
if (global.__electronAppPaths) {
  const { appData } = global.__electronAppPaths;
  config = {
    ...baseConfig,
    excelPath: path.join(appData, 'data', 'tasks-dramabagus.xlsx'),
    browser: {
      profileDir: path.join(appData, 'browser-data'),
      browsersPath: path.join(appData, 'pw-browsers'),
    },
    validation: { screenshotDir: path.join(appData, 'screenshots') },
  };
}
```

### 3.2 暂停与日志注入（lib/utils.js）

**问题**：原脚本用 `readline` 等待用户按 Enter，Electron 无终端。

**实现**：增加 `setPauseHandler` / `setLogHandler`，Electron 启动时注入弹窗和 UI 日志：

```javascript
// utils.js
let customPauseHandler = null;
let customLogHandler = null;

function setPauseHandler(handler) { customPauseHandler = handler; }
function setLogHandler(handler) { customLogHandler = handler; }

function pauseForUser(message) {
  if (customPauseHandler) return customPauseHandler(message);
  // 原有 readline 逻辑...
}

function log(message, level) {
  const line = `[${time}] ${prefix}  ${message}`;
  if (customLogHandler) customLogHandler(line, level);
  console.log(line);
}
```

```javascript
// electron-main.js 中，用户点击「开始运行」时
setPauseHandler((message) => {
  return new Promise(resolve => {
    pauseResolve = resolve;
    mainWindow.webContents.send('show-pause', message);
  });
});
setLogHandler((line) => mainWindow.webContents.send('log', line));
```

### 3.3 主逻辑入口分离（index-electron.js）

**原因**：原 `index.js` 含 `process.exit(1)`，Electron 中不宜直接退出进程。

**实现**：将 `main()` 逻辑抽到 `index-electron.js`，导出 `run()`，由 Electron 的 IPC handler 调用。

### 3.4 首次运行初始化

- 创建 `appData/data`、`browser-data`、`pw-browsers`、`screenshots` 目录
- 若 `data/tasks-dramabagus.xlsx` 不存在，从打包资源复制或现场生成

## 五、打包步骤

### 5.1 安装依赖

```bash
npm install
npm run setup   # 安装 Playwright Chromium（开发/测试用）
```

### 5.2 打包

```bash
# Windows 安装程序
npm run build:win
# 输出：dist/DramaBagus批量广告工具 Setup 1.0.0.exe

# Mac 安装程序（支持 Intel + Apple Silicon）
npm run build:mac
# 输出：dist/DramaBagus批量广告工具-1.0.0-universal.dmg

# 仅生成未打包目录（快速测试）
npm run pack:win
npm run pack:mac
```

### 5.3 本地测试

```bash
npm run electron
```

## 七、注意事项

### 7.1 Chromium 未打包

- 在 Mac 上打 Windows 包时，无法打包 Windows 版 Chromium
- 首次运行需联网，Playwright 会自动下载 Chromium（约 150MB）
- 若需离线完整版，需在 Windows 上执行 `npm run build:win`

### 7.2 跨平台打包

- Mac 可打 Windows 包（`--win`）
- Windows 可打 Mac 包（需 macOS 环境或 CI）
- 各平台安装包互不通用

### 7.3 代码签名（Mac）

- 未配置开发者证书时，打包会提示「skipped macOS application code signing」
- 用户首次打开需右键 →「打开」以绕过 Gatekeeper

## 八、迁移到其他项目

若要将类似 Node.js + Playwright 项目打包，可按以下步骤：

1. **安装依赖**：`npm i -D electron electron-builder`
2. **修改 package.json**：设置 `main`、添加 `build` 配置和脚本
3. **创建 Electron 三件套**：`electron-main.js`、`electron-ui.html`、`electron-preload.js`
4. **抽取主逻辑**：新建 `index-electron.js` 导出 `run()`
5. **适配 utils**：增加 `setPauseHandler` / `setLogHandler`，在 `pauseForUser` 和 `log` 中调用
6. **适配 config**：检测 `global.__electronAppPaths`，使用 `appData` 路径
7. **首次运行**：在 electron-main 中创建目录、复制/生成模板文件

## 九、常见问题

| 问题 | 处理 |
|------|------|
| `npm install` 报 EPERM | 关闭沙盒或使用 `required_permissions: ["all"]` |
| 打包后 exe 无法运行 | 检查是否缺少 `files` 中的关键文件 |
| 首次运行卡住 | 确认网络正常，用于下载 Chromium |
| Mac 提示无法打开 | 右键应用 → 选择「打开」 |
