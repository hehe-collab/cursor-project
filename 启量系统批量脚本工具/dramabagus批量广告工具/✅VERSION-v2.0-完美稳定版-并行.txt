═══════════════════════════════════════════════════════════

### 1. 生成Excel模板
```bash
cd /Volumes/存钱罐/cursor项目/ad-automation
npm run template
```

### 2. 填写任务数据
```
任务ID | 广告系列ID | 账户ID | 广告名称 | 投放目标 | 出价 | 预算 | 素材ID | 素材关键词
------|-----------|--------|---------|---------|------|------|--------|----------
1     | 12345...  | 7560... | 测试广告 | 价值     | 1.2  | 1000 | 1234,5678 | 
2     | 12345...  | 7560... | 测试广告2| 转化     | 1.1  | 500  |        | 关键词
```

**字段说明**：
- 素材ID：多个用逗号/分号/空格/换行分隔
- 素材关键词：只在素材ID为空时使用
- 投放目标：价值 或 转化
- 出价/预算：会自动验证

```bash
npm start
```

1. 读取Excel任务，显示风险控制验证
2. 启动浏览器，等待手动登录
3. 确认登录后按 Enter 开始
4. 自动创建10个标签页（并行模式）
5. 预热素材弹窗（解决第1个账户问题）
6. 所有标签页同时执行广告创建
7. 关键步骤自动截图
8. 任务完成，显示统计结果

═══════════════════════════════════════════════════════════
🔧 开发历程（重要里程碑）
═══════════════════════════════════════════════════════════

### 问题1：第一个账户素材选择总是失败 ❌
**现象**：每个任务的第1个账户，素材ID搜索后数量对不上
**原因**：素材弹窗首次打开时，Vue组件未完全初始化
**解决方案**：
- 方案A：第1个账户等待1500ms，其他200ms ✅
- 方案B：预热机制 - 任务开始前预先打开并关闭弹窗 ✅（最终采用）
- 方案C：激活搜索输入框 - 先输入空格再清空 ✅（最终采用）

### 问题2：素材关键词检索选错素材 ❌
**现象**：关键词检索时，如果没有匹配的素材，会选择其他不相关素材
**原因**：关键词检索返回结果为空时，页面仍显示其他素材
**解决方案**：优先使用素材ID检索，关键词作为备用 ✅

### 问题3：风险控制需求 ⚠️
**需求**：防止出价或预算过高，导致账户"跑炸"
**解决方案**：
- 价值目标：出价 ≥ 1.1
- 转化目标：出价 ≤ 1.3
- 预算：50 ~ 5000
- Excel读取时预验证 + 提交前二次验证 ✅

### 问题4：并行模式第一个标签页不动 ❌
**现象**：10个标签页，第1个不执行，后面9个正常
**原因**：`preparePage` 在并行模式下被跳过，导致第1个页面未准备好
**解决方案**：
- 无论串行还是并行，都先调用 `preparePage`
- 并行模式下，新标签页继承登录状态 ✅

### 问题5：页面重复刷新两次 ❌
**现象**：并行模式下，每个任务的页面会刷新2次
**原因**：
- `parallel.js` 创建标签页时已加载页面（第1次）
- `executeTask` 中 `if (taskIndex > 0)` 再次刷新（第2次）
**解决方案**：
- `executeTask` 增加 `isParallel` 参数
- 并行模式下跳过刷新逻辑 ✅

### 问题6：策略1总是失败，日志噪音 ⚠️
**现象**：日志中总是显示"策略1失败，尝试策略2..."
**原因**：策略1选择器不准确，策略2 100%成功
**解决方案**：
- 移除策略1和策略3
- 直接使用策略2
- 日志清爽30% ✅

### 1. 素材ID激活机制
```javascript
// 解决Vue组件未初始化问题
await materialIdInput.click();
await materialIdInput.fill(' ');     // 激活组件
await page.waitForTimeout(100);
await materialIdInput.fill('');      // 清空
await page.waitForTimeout(100);
await materialIdInput.fill(ids);     // 真正输入
```

### 2. 预热弹窗机制
```javascript
// 在第1个任务前，预先打开并关闭素材弹窗
// 让Vue组件完成初始化
if (isFirstTask) {
  await prepareMaterialDialog(page);
}
```

### 3. 风险控制双重验证
```javascript
// 第一重：Excel读取时
validateTaskData(task);

// 第二重：提交前
validateBeforeSubmit(page, task);
```

### 4. 并行模式区分
```javascript
// 串行：需要刷新页面
executeTask(page, task, index, total, isParallel=false)

// 并行：不需要刷新页面
executeTask(page, task, index, total, isParallel=true)
```

### 5. 智能延迟
```javascript
// 第1个账户：需要更多时间
delay.materialDialogOpen: 900ms

// 其他账户：已经预热，速度快
delay.materialDialogOpenOthers: 200ms
```
