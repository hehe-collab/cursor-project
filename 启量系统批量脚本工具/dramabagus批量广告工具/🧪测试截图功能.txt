三、故障排查
--------------------------------------------------------------------

### 问题1：没有生成截图

**检查1**：确认配置已开启
```bash
# 查看config.js
cat config.js | grep enableScreenshot
```

应该显示：
```javascript
enableScreenshot: true,
```

---

**检查2**：确认目录权限
```bash
# 检查截图目录是否可写
ls -ld ./screenshots/
```

如果目录不存在，手动创建：
```bash
mkdir -p ./screenshots/
```

---

**检查3**：查看终端错误
- 如果有 ⚠️ 截图失败 的警告，查看具体错误信息
- 如果错误是 "quality is unsupported"，说明修复未生效

---

### 问题2：截图是黑屏或空白

**原因**：页面还未完全加载就截图了

**解决方法**：增加等待时间
```javascript
// lib/automation.js - takeScreenshot()
await page.waitForTimeout(500); // 截图前等待500ms
await page.screenshot({...});
```

---

### 问题3：截图文件很小（<10KB）

**原因**：截图损坏或页面异常

**解决方法**：
1. 检查浏览器是否正常显示
2. 检查page对象是否已关闭
3. 增加错误日志：
```javascript
log(`  截图文件大小: ${fs.statSync(filepath).size} bytes`);
```

---

### 问题4：10并发时截图混乱

**原因**：多个任务同时截图，文件名冲突

**当前方案**：
- 每个任务有独立文件夹（按任务ID）
- 文件名包含时间戳（毫秒级）
- 理论上不会冲突

**如果仍有问题**：
- 检查任务ID是否重复
- 检查时间戳是否正确生成
