Phase 2：并行优化方案设计
设计时间：2026-02-10
目标：在多个浏览器标签页中同时处理不同任务，大幅缩短总耗时

一、性能对比分析
--------------------------------------------------------------------

【当前性能 - 串行执行】
- 单个任务耗时：约2.5-3分钟
- 10个任务耗时：约25-30分钟
- 瓶颈：只使用1个标签页，CPU和网络资源利用率低

【优化后 - 并行执行（5并发）】
- 单个任务耗时：不变（2.5-3分钟）
- 10个任务耗时：约6-8分钟（5个任务一批，分2批）
- 提升：**时间缩短到原来的 25-30%**

【硬件能力评估】
设备：Mac Mini M4 Pro
- CPU：10核（性能核+效率核）
- 内存：16-32GB
- 网络：千兆

**评估**：
✅ 支持5-6个并发标签页（稳健）
✅ 支持8-10个并发（激进，但可能不稳定）
✅ 建议：5个并发（平衡性能与稳定性）


二、并行架构设计
--------------------------------------------------------------------

### 核心思路

```
当前架构（串行）：
┌──────────┐
│ 浏览器    │
│  ├─ Tab1 │ → 任务1 → 任务2 → 任务3 → ... → 任务10
└──────────┘

优化架构（并行）：
┌──────────┐
│ 浏览器    │
│  ├─ Tab1 │ → 任务1 \
│  ├─ Tab2 │ → 任务2  \
│  ├─ Tab3 │ → 任务3   }  同时执行
│  ├─ Tab4 │ → 任务4  /
│  ├─ Tab5 │ → 任务5 /
│  │        │
│  │  等第一批完成...
│  │        │
│  ├─ Tab1 │ → 任务6 \
│  ├─ Tab2 │ → 任务7  \
│  ├─ Tab3 │ → 任务8   }  第二批
│  ├─ Tab4 │ → 任务9  /
│  ├─ Tab5 │ → 任务10/
└──────────┘
```

### 实现策略

#### 方案1：多标签页 + Promise.all（推荐）✅

**原理**：
```javascript
// 1. 启动浏览器
const browser = await chromium.launchPersistentContext(...)

// 2. 准备任务队列
const tasks = [...] // 读取所有任务

// 3. 分批并行执行
const batchSize = 5;
for (let i = 0; i < tasks.length; i += batchSize) {
  const batch = tasks.slice(i, i + batchSize);
  
  // 4. 为每个任务创建新标签页
  const promises = batch.map(async (task, index) => {
    const page = await browser.newPage();
    await page.goto(taskUrl);
    await executeTaskGroup(page, task, index);
    await page.close();
  });
  
  // 5. 等待这一批全部完成
  await Promise.all(promises);
  
  // 6. 批次间短暂延迟（避免服务器压力）
  await delay(2000);
}
```

**优点**：
✅ 实现简单，代码改动小
✅ 可以复用现有的 executeTaskGroup 函数
✅ 每个标签页独立，不会互相干扰
✅ 错误处理清晰（某个标签页失败不影响其他）

**缺点**：
❌ 需要管理多个标签页
❌ 内存占用会增加（5个标签页 ≈ 1-2GB额外内存）

---

#### 方案2：多浏览器实例（备选）

**原理**：
```javascript
// 为每个任务启动独立的浏览器
const browsers = await Promise.all(
  batch.map(() => chromium.launchPersistentContext(...))
);
```

**优点**：
✅ 隔离性更好

**缺点**：
❌ 资源占用大（每个浏览器实例 ≈ 500MB+）
❌ 启动慢
❌ 复杂度高

**结论**：不推荐

--------------------------------------------------------------------

### Step 1：配置文件调整

**config.js**：
```javascript
// ===== 并发配置 =====
parallel: {
  enabled: false,          // 是否启用并行（默认关闭）
  maxConcurrent: 5,        // 最大并发数

},
```

---

### Step 2：任务分组函数

**lib/parallel.js**（新文件）：
```javascript
/**
 * 将任务数组分成批次
 * @param {Array} tasks - 所有任务
 * @param {number} batchSize - 每批任务数量
 * @returns {Array<Array>} - 分批后的任务数组
 */
function splitIntoBatches(tasks, batchSize) {
  const batches = [];
  for (let i = 0; i < tasks.length; i += batchSize) {
    batches.push(tasks.slice(i, i + batchSize));
  }
  return batches;
}

/**
 * 并行执行一批任务
 * @param {BrowserContext} browser - 浏览器上下文
 * @param {Array} batch - 一批任务
 * @param {number} batchIndex - 批次序号
 */
async function executeTaskBatch(browser, batch, batchIndex) {
  log(`\n╔══════════════════════════════════════════════════╗`);
  log(`║  执行第 ${batchIndex} 批任务（${batch.length} 个并发）     ║`);
  log(`╚══════════════════════════════════════════════════╝\n`);

  // 创建Promise数组
  const promises = batch.map(async (taskGroup, index) => {
    try {
      // 为每个任务创建新标签页
      const page = await browser.newPage();
      
      // 导航到任务页面
      await page.goto(config.taskUrl);
      await page.waitForLoadState('domcontentloaded');
      
      // 执行任务
      const taskNumber = batchIndex * batch.length + index + 1;
      await executeTaskGroup(page, taskGroup, taskNumber);
      
      // 关闭标签页
      await page.close();
      
      return { success: true, taskId: taskGroup.taskId };
    } catch (error) {
      log(`❌ 任务 [${taskGroup.taskId}] 执行失败: ${error.message}`, 'ERROR');
      return { success: false, taskId: taskGroup.taskId, error: error.message };
    }
  });

  // 等待所有任务完成
  const results = await Promise.all(promises);
  
  return results;
}

module.exports = {
  splitIntoBatches,
  executeTaskBatch,
};
```

---

### Step 3：主流程调整

**index.js**：
```javascript
const { executeTaskGroup } = require('./lib/automation');
const { splitIntoBatches, executeTaskBatch } = require('./lib/parallel');

async function main() {
  // ... 读取任务 ...
  
  if (config.parallel.enabled) {
    // 【并行模式】
    log('\n🚀 并行模式已启用');
    log(`📋 并发数: ${config.parallel.maxConcurrent}`);
    log(`📋 任务总数: ${taskGroups.length}`);
    
    // 分批
    const batches = splitIntoBatches(taskGroups, config.parallel.maxConcurrent);
    log(`📋 分为 ${batches.length} 批执行\n`);
    
    let allResults = [];
    
    // 执行每一批
    for (let i = 0; i < batches.length; i++) {
      const batchResults = await executeTaskBatch(browser, batches[i], i + 1);
      allResults = allResults.concat(batchResults);
      
      // 批次间延迟（避免服务器压力）
      if (i < batches.length - 1) {
        log(`\n⏳ 等待 ${config.parallel.batchDelay}ms 后执行下一批...\n`);
        await delay(config.parallel.batchDelay);
      }
    }
    
    // 统计结果
    const successCount = allResults.filter(r => r.success).length;
    const failedCount = allResults.filter(r => !r.success).length;
    
    log('\n╔══════════════════════════════════════════════════╗');
    log('║                  执行完毕（并行）                 ║');
    log('╚══════════════════════════════════════════════════╝');
    log(`✅  成功: ${successCount} 个`);
    if (failedCount > 0) {
      log(`❌  失败: ${failedCount} 个`, 'ERROR');
    }
    log(`📋  总计: ${taskGroups.length} 个\n`);
    
  } else {
    // 【串行模式】（原有逻辑）
    for (let i = 0; i < taskGroups.length; i++) {
      await executeTaskGroup(page, taskGroups[i], i + 1);
    }
  }
  
  // ... 结束 ...
}
```


四、风险控制与稳定性保障
--------------------------------------------------------------------

### 1. 服务器压力控制

**问题**：5个标签页同时提交请求，服务器可能拒绝或限流

**解决方案**：
```javascript
// 批次间延迟
config.parallel.batchDelay = 2000;  // 2秒

// 任务内部的提交也有2秒间隔（已有）
```

---

### 2. 内存管理

**问题**：5个标签页同时运行，内存占用增加

**监控**：
```javascript
// 在批次开始前检查内存
const memUsage = process.memoryUsage();
if (memUsage.heapUsed > 1.5 * 1024 * 1024 * 1024) { // 超过1.5GB
  log('⚠️  内存占用过高，建议降低并发数', 'WARN');
}
```

**清理**：
```javascript
// 每批结束后强制GC（如果可用）
if (global.gc) {
  global.gc();
}
```

---

### 3. 错误隔离

**问题**：某个任务失败不应影响其他任务

**解决方案**：
```javascript
// 每个任务独立 try-catch
const promises = batch.map(async (task) => {
  try {
    // 执行任务...
  } catch (error) {
    // 记录错误，但不中断其他任务
    return { success: false, error };
  }
});

// Promise.all 确保等待所有任务（无论成功失败）
await Promise.all(promises);
```

---

### 4. 超时保护

**问题**：某个任务卡住可能拖累整批

**解决方案**：
```javascript
// 为每个任务设置超时
const promises = batch.map(async (task) => {
  return Promise.race([
    executeTaskGroup(page, task, index),
    new Promise((_, reject) => 
      setTimeout(() => reject(new Error('任务超时')), 10 * 60 * 1000) // 10分钟超时
    )
  ]);
});
```

六、性能预估
--------------------------------------------------------------------

【测试场景】：10个任务，每个任务2.5分钟

| 模式       | 并发数 | 批次 | 每批耗时 | 批次间延迟 | 总耗时      | 提升     |
|------------|--------|------|----------|------------|-------------|----------|
| 串行       | 1      | 10   | 2.5分钟  | 0秒        | 25分钟      | 基准     |
| 并行（2）  | 2      | 5    | 2.5分钟  | 2秒        | 12.7分钟    | **50%**  |
| 并行（3）  | 3      | 4    | 2.5分钟  | 2秒        | 10.2分钟    | **60%**  |
| 并行（5）✅| 5      | 2    | 2.5分钟  | 2秒        | 5.0分钟     | **80%**  |
| 并行（8）  | 8      | 2    | 2.5分钟  | 2秒        | 5.0分钟     | **80%**  |

**结论**：
- **5并发**是最优选择：稳定且效率高
- 8并发提升不大（仍然受限于批次数）
- **建议**：默认5并发，保守可用3并发
