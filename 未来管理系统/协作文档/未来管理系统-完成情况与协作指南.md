## 2. 启动方式（本地必须）

在 **`未来管理系统`** 目录：

```bash
npm install    # 根目录、frontend；迁移脚本请在 backend-java/scripts 内 npm install
npm run start  # 同时启动：Java backend（3001）+ 前端（5173）
```

- 浏览器访问：**http://localhost:5173**（看板：**/dashboard**）  
- **指令 #056**：**仅 Java + 前端**；**`frontend/vite.config.js`** 将 **`/api` 全部代理到 3001**。  
- **JWT**：由 **Java** **`JwtAuthenticationFilter`** 校验（**`GET /api/uploads/`** 静态封面除外规则不变）。  
- **未启动 Java（3001）时**，浏览器会出现 **ERR_CONNECTION_REFUSED** 或接口失败。

默认管理员：**admin** / **admin123**（首次登录后应改密）。

**Java 后端演示数据（可选）**：在已导入 **`schema.sql`** 的前提下，执行 **`backend-java/scripts/import-demo-data.sh`**（或 **`mysql -u root … < backend-java/src/main/resources/sql/demo_data.sql`**），详见 **`backend-java/README.md`**「导入演示数据」。

**重要说明：**

- 脚本支持**重复执行**（会先按 `demo_data.sql` 清理同批演示行再插入；含 **`drama_episodes`** 显式清理，避免唯一键冲突）。
- 如遇 **`ERROR 1045`**（Access denied），请核对 MySQL **`root`** 密码：与 Spring **`application.yml` / `MYSQL_PASSWORD`** 一致；无密码时不要使用 **`mysql -p`**。
- 建议验收：连续执行 **`bash scripts/import-demo-data.sh` 两次**，均应成功（详见 **§15.1**）。

### ✅ 指令 #088：页面布局占比不一致 — 原因分析（已完成，2026-04-08）

- **结论（无代码整改，仅分析）**：占比不一致来自 **(1) 列表页表格高度统一用 `calc(100vh - Npx)` 且 N 各页不同**、**(2) 子页面已处于 `Layout` 的 `el-main`  flex 剩余高度内，仍按视口全高扣减，造成与父级实际高度脱节**、**(3) 部分页额外 `padding`/`page-list-layout` 使用不一致**、**(4) 看板 `Dashboard.vue` 单独使用 `calc(100vh - 48px - 36px - 24px)` 与全局列表策略不一致**。  
- **主布局**：**`frontend/src/views/Layout.vue`** — **`height: 100vh`**；**侧栏** **`210px`/`64px`**；**`el-header`****`48px`**；**`tabs-bar`****`min-height: 36px`**；**`el-main.main`****`flex: 1`**、**`padding: var(--page-padding, 12px)`**（**`compact-theme.css`** 中 **`--page-padding: 12px`**）；**`.main-router-host > *`** **`flex: 1 1 auto`**。  
- **列表通用**：**`compact-theme.css`** **`.page-list-layout`** — **`flex: 1`**、**`padding: 8px`**、纵向 flex；**`User.vue`** 仅该类，**表格 `height="calc(100vh - 220px)"`**。  
- **充值**：**`Recharge.vue`** **`page-list-layout`** + **scoped** **`.recharge-container { padding: 20px }`**（与 **8px** 列表基线不一致）；**表格外层有 `el-tabs`**，**表格 `calc(100vh - 280px)`**，**N 更大 → 表格可视更矮**。  
- **短剧**：**`DramaManage.vue`** **`page-list-layout`** + **`.drama-manage-container { padding: 20px }`**；**表格 `calc(100vh - 320px)`**。  
- **标题包**：**`TitlePack.vue`** **未使用** **`page-list-layout`**（**`title-pack-page` 单类**），**表格 `calc(100vh - 300px)`**，**未吃满 `main-router-host` flex**。

- **全局**：**`App.vue`** **`* { margin:0;padding:0;box-sizing:border-box }`**，**`html,body,#app { height:100% }`**；**无**独立 **`main.css`**（以 **`App.vue` + `compact-theme.css`** 为准）。  
- **后续整改方向（供 #089）**：表格高度改为 **父级 `100%`/`flex:1` + 卡片内 flex**，或 **CSS 变量统一 `--layout-chrome-height`**；**`TitlePack` 补** **`page-list-layout`**；**收敛** **`20px` vs `8px` padding**。

## 15. 常见问题与解决方案（Java 后端 / 演示数据）

### 15.1 演示数据导入

#### `ERROR 1062 (23000): Duplicate entry '501-1' for key 'drama_episodes.uk_drama_episode'`

**根因**：**`FOREIGN_KEY_CHECKS = 0`** 下删除 **`dramas`** 时子表 **`drama_episodes`** 可能残留，再次 **`INSERT`** 触发 **`uk_drama_episode (drama_id, episode_num)`**。

**处理**：

1. 使用仓库内**最新** **`demo_data.sql`**（清理段已含 **`DELETE FROM drama_episodes WHERE drama_id BETWEEN 501 AND 505`**）。
2. 或手动执行后再导入：  
   `DELETE FROM drama_episodes WHERE drama_id BETWEEN 501 AND 505;`  
   （必要时再删 **`dramas`** 等同批演示行，见脚本内顺序。）

**验证**：同一终端连续执行 **`bash scripts/import-demo-data.sh` 两次**，均应 **演示数据导入成功**。

#### `ERROR 1045 (28000): Access denied for user 'root'@'localhost'`

**根因**：**`mysql -u root -p`** 交互输入的密码与 MySQL **`root`** 实际凭证不一致；或 **`root`** 本机**无密码**却使用了 **`-p`**。

**处理**：

- 与 Spring Boot 一致时：先 **`export MYSQL_PASSWORD='实际密码'`**，再 **`bash scripts/import-demo-data.sh`**（脚本会带 **`-p"$MYSQL_PASSWORD"`**）。
- **无密码**：**不要**加 **`-p`**，使用 **`mysql -u root drama_system < ...`**。

### 15.2 Java 接口快速验收（演示数据已导入、后端 **3001** 已启动）

```bash
TOKEN=$(curl -s -X POST http://localhost:3001/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}' | jq -r '.data.token')

echo "=== 短剧列表 total ==="
curl -s "http://localhost:3001/api/dramas?page=1&pageSize=5" \
  -H "Authorization: Bearer $TOKEN" | jq '.data.total'

echo "=== 短剧 501 剧集条数 ==="
curl -s "http://localhost:3001/api/dramas/501" \
  -H "Authorization: Bearer $TOKEN" | jq '.data.episodes | length'

echo "=== 用户列表 total ==="
curl -s "http://localhost:3001/api/users?page=1&pageSize=10" \
  -H "Authorization: Bearer $TOKEN" | jq '.data.total'

echo "=== 充值统计 ==="
curl -s "http://localhost:3001/api/recharge/stats" \
  -H "Authorization: Bearer $TOKEN" | jq '.data'
```

**期望**：短剧 **`total`** 为 **5**；**501** 的 **`episodes`** 长度为 **2**（当前 **`demo_data.sql`** 仅为剧 **501** 插入 **2 集**）；用户 **`total`** 为 **10**；充值 **`total_count`** 为 **20**、**`pending_count`** 为 **3**。

### 15.3 API 报 500 或 Vite 报 `ECONNREFUSED`（仅 Java 时代）

**现象**：前端页面接口失败；Vite 终端 **`http proxy error` … `ECONNREFUSED`**。

**常见根因**：

1. **仅启动了前端**，**Java（3001）未监听**。  
   **处理**：根目录 **`npm run start`**，或 **`cd backend-java && source scripts/use-java17.sh && mvn spring-boot:run`**。

2. **3001 被占用**导致 Spring 启动失败。  
   **处理**：**`lsof -ti:3001 | xargs kill -9`** 后重试。

3. **401**：Token 无效或过期；重新登录。**`JWT_SECRET`** 需在 **`backend-java`** 运行环境中与签发时一致。
