# 短剧剧集合并工具 - 需求与对话记录

> **记录说明**：本文档用于记录“短剧剧集合并工具”的开发思路、需求讨论以及对话要点，充当需求说明书并持续更新。

## 核心目标
将一部分成几十集的短剧无损、快速合并成一整个视频，并自动切除每集片尾的多余画面（如高光、停顿、引导继续观看等）。

---

## [2026-03-13] 构思与初步规划

**讨论要点**：
1. **裁剪点定位方案**：探讨了固定时长裁剪、黑场静音检测、图像匹配、OCR文字识别等多种方案。
2. **极速无损合并机制**：明确必须采用 FFmpeg 的 Stream Copy（`-c copy`）和 Concat Demuxer 进行流拼接，避免重新编码（Re-encode），从而实现秒级的高画质无损合并。
3. **工作流设计**：读取剧集列表并自然排序 -> 逐集分析裁剪点并截断生成中间文件 -> Concat无损拼接 -> 自动清理中间生成的零碎文件。

---

## [2026-03-13] 确定核心方案与交互要求

**用户反馈与需求更新**：
- **方案确立**：经过用户观察，该批短剧每集结尾冗余时间基本一致，因此果断采用**方案A（固定倒数时长截断法）**。简单、轻量且足够满足需求。
- **重点功能要求**：脚本必须加入**手动输入剪切时间**的功能选项。由于不同剧集之间可能存在秒数差异，通过人工观察后输入参数，让程序执行重复劳动。
- **文档要求**：要求在当前文件夹中记录本次及后续对话的全文内容并持续自动更新，作为长期的参考资料。

**AI 响应与思路扩展**：
1. **交互逻辑设计**：脚本启动时，可以通过交互式问答（Terminal Prompt）让用户输入：`请输入需要切除片尾的秒数（例如 8）：`，或者支持更快捷的命令行参数 `python merge.py --trim-end 8`。
2. **细节考虑（例外情况）**：考虑到部分剧集可能有特殊性（比如“最后一集不需要切除片尾”，或者“是否需要保留最后一集的彩蛋？”），在后续开发中可以加入一个小小的询问：“是否对最后一集也执行相同的裁切逻辑？”来提高容错率。
3. **记录落实**：已建立此文档并同步全局对话记录，未来所有变动都将追溯于此。

## [2026-03-13] 开始编码与实现

**用户反馈**：对交互逻辑（启动脚本 -> 输入秒数 -> 确认大结局是否保留 -> 瞬间出片）表示完全认同，指示开始编码。

**技术实现路径**：
1. **基础环境**：新建 `requirements.txt`。虽然大部分操作可以通过原生的 `subprocess` 直接调用系统安装的 `ffmpeg`，但引入 `ffmpeg-python` 或其他简单的自然排序库可以让代码更清晰。
2. **核心代码 `merge_drama.py`**：
   - 使用正则表达式进行自然排序（确保 `1.mp4`, `2.mp4` ... `10.mp4` 顺序正确，而非 `1, 10, 2`）。
   - 用户交互：使用 `input()` 获取切除秒数和最后一集的处理意向。
   - 时长探测：使用 `ffprobe` 提取视频时长，减去设定秒数得到截断点。
   - 裁切与合并：生成 `concat.txt`，批量流拷贝生成中间文件，再合并。最后自动清理残余文件。
3. **易用性**：编写一个 `run.sh` 脚本供 macOS/Linux 用户一键运行。

## [2026-03-13] 需求迭代：指定路径读取与智能命名

**用户反馈**：希望不再频繁复制文件到 input 文件夹，而是能够指定读取路径；并且希望合成后的文件名自动使用剧集所在的文件夹名称。

**构思与规划（不改代码阶段）**：
1. **输入方式重构（指定路径+拖拽）**：
   - 脚本启动后的第一步，不再是直接扫描当前目录，而是提示：`👉 请输入或拖入短剧文件夹路径：`。
   - Mac/终端 的特性是，如果你把一个文件夹直接拖进终端窗口，它会自动把这个文件夹的“绝对路径”填进去。这比手动打字输入路径要方便 100 倍。
2. **智能提取文件夹名**：
   - 如果用户拖入的路径是 `/Volumes/硬盘/我的短剧/霸道总裁爱上我_70集全`。
   - 脚本会自动提取最后一段名字 `霸道总裁爱上我_70集全`。
   - 自动生成输出文件名：`霸道总裁爱上我_70集全.mp4`。不需要用户再手动输入命名。
3. **输出位置探讨**：
   - 既然文件不放在 input 里了，那输出的成品是统一存放在工具本身的 `output/` 文件夹里，还是直接在原剧集文件夹旁边生成？通常建议集中放在工具的 `output/` 文件夹下，保持源文件目录的干净。这可以后续由用户拍板。

**用户选择**：确认上述方案，并选择了 **方案B**，即将合成的成品统一导出并管理在工具目录下的 `output/` 文件夹中。开始执行代码修改。
